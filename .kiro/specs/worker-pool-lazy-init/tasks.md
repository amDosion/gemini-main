# Implementation Plan

- [ ] 1. 添加状态枚举和核心属性
  - [ ] 1.1 创建 `PoolStatus` 枚举类
    - 定义 IDLE, STARTING, RUNNING, ERROR, STOPPED 状态
    - 添加状态描述方法
    - _Requirements: 4.1_
  - [ ]* 1.2 编写 Property 4 属性测试
    - **Property 4: Status Enum Validity**
    - **Validates: Requirements 4.1**
  - [ ] 1.3 修改 `UploadWorkerPool.__init__` 添加新属性
    - 添加 `_status: PoolStatus`
    - 添加 `_init_lock: asyncio.Lock`
    - 添加 `_redis_connected: bool`
    - 添加 `_last_error: Optional[str]`
    - 添加 `_started_at: Optional[datetime]`
    - 添加 `_init_retry_count: int`
    - _Requirements: 1.1, 3.1_

- [ ] 2. 实现懒加载核心方法
  - [ ] 2.1 实现 `ensure_started()` 方法
    - 使用 `_init_lock` 保证并发安全
    - 检查当前状态，RUNNING 直接返回
    - STARTING 状态等待完成
    - IDLE/ERROR 状态触发启动
    - _Requirements: 1.3, 3.1, 3.2, 5.3_
  - [ ]* 2.2 编写 Property 3 属性测试
    - **Property 3: Concurrent Initialization Safety**
    - **Validates: Requirements 3.1, 3.2**
  - [ ] 2.3 实现 `check_pending_tasks()` 方法
    - 查询数据库中 status='pending' 或 'uploading' 的任务
    - 返回是否存在未完成任务
    - _Requirements: 2.1_
  - [ ] 2.4 实现 `get_status()` 方法
    - 返回状态信息字典
    - 包含 status, worker_count, message, last_error, started_at
    - _Requirements: 4.1, 4.2, 4.3_

- [ ] 3. 修改启动流程
  - [ ] 3.1 修改 `start()` 方法支持状态转换
    - 启动前设置状态为 STARTING
    - 成功后设置状态为 RUNNING
    - 失败时设置状态为 ERROR
    - 记录 `_started_at` 时间
    - _Requirements: 1.3, 5.1, 5.2_
  - [ ]* 3.2 编写 Property 1 属性测试
    - **Property 1: Lazy Start Timing**
    - **Validates: Requirements 1.3**
  - [ ] 3.3 实现 `_connect_redis_only()` 方法
    - 仅建立 Redis 连接，不启动 Workers
    - 用于健康检查和状态查询
    - _Requirements: 1.2_

- [ ] 4. Checkpoint - 确保所有测试通过
  - Ensure all tests pass, ask the user if questions arise.

- [ ] 5. 修改应用生命周期
  - [ ] 5.1 修改 `lifespan()` 函数
    - 启动时调用 `check_pending_tasks()`
    - 有未完成任务时调用 `start()`
    - 无未完成任务时保持 IDLE
    - _Requirements: 2.1, 2.3_
  - [ ]* 5.2 编写 Property 2 属性测试
    - **Property 2: Auto-Start on Pending Tasks**
    - **Validates: Requirements 2.1, 2.2**
  - [ ] 5.3 修改健康检查端点
    - 添加 `worker_pool` 字段到响应
    - 包含 status, worker_count, message
    - _Requirements: 4.1, 4.2, 4.3_

- [ ] 6. 修改任务入队逻辑
  - [ ] 6.1 修改 `enqueue_upload_task()` 函数
    - 调用 `ensure_started()` 确保 Worker 池已启动
    - 处理启动失败的情况
    - _Requirements: 1.3, 5.3_
  - [ ] 6.2 修改 storage router 中的上传接口
    - 在创建任务后调用 `ensure_started()`
    - 添加错误处理
    - _Requirements: 1.3_

- [ ] 7. 实现错误恢复机制
  - [ ] 7.1 实现重试逻辑
    - 最大重试次数 3 次
    - 指数退避（1s, 2s, 4s）
    - 超过重试次数返回错误
    - _Requirements: 3.3, 5.1, 5.2_
  - [ ]* 7.2 编写错误恢复单元测试
    - 测试 Redis 连接失败后重试
    - 测试 Worker 启动失败后重试
    - _Requirements: 5.1, 5.2, 5.3_

- [ ] 8. Final Checkpoint - 确保所有测试通过
  - Ensure all tests pass, ask the user if questions arise.
