# Virtual Try-On 使用说明

## 功能概述

Virtual Try-On（虚拟试衣）是一个基于 Gemini 2.5 和 Imagen 3 的 AI 服装替换功能，可以在人物照片中智能替换服装。

---

## 完整工作流程

### 步骤 1：上传人物照片

1. 点击界面左下角的 **"Virtual Try-On"** 模式
2. 点击输入框左侧的 **📎 附件按钮**
3. 选择一张清晰的人物照片（建议全身或半身照）
4. 照片会显示在画布中央

**照片要求**：
- 格式：JPG、PNG
- 尺寸：建议 512×512 到 2048×2048
- 内容：人物清晰可见，服装完整

---

### 步骤 2：选择替换区域

在右侧控制面板中选择要替换的服装类型：

| 选项 | 说明 | 适用场景 |
|------|------|----------|
| **上衣** | 替换上半身服装 | T恤、衬衫、夹克、外套 |
| **下装** | 替换下半身服装 | 裤子、裙子、短裤 |
| **全身** | 替换整套服装 | 连衣裙、套装 |

---

### 步骤 3：描述想要的服装

在输入框中详细描述想要替换的服装：

**✅ 好的描述示例**：
```
一件深绿色羽绒服，橙色内衬，里面穿白色圆领T恤
```

```
黑色皮夹克，自然光照，柔和阴影
```

```
正式的深蓝色西装外套，专业风格，锐利的折痕
```

**❌ 不好的描述**：
```
夹克
```

```
好看的衣服
```

---

### 步骤 4：生成试衣效果

1. 点击 **发送按钮** 或按 **Enter**
2. AI 会自动：
   - 使用 Gemini 2.5 识别并分割服装区域
   - 使用 Imagen 3 生成替换后的图像
3. 等待 10-30 秒（取决于图像大小）
4. 结果会显示在画布中央

---

### 步骤 5：查看和对比结果

**查看结果**：
- 生成的图像会自动显示在画布中央
- 使用鼠标滚轮可以**缩放**查看细节
- 按住鼠标左键可以**拖动**画布

**对比原图**：
1. 点击右下角的 **"对比"** 按钮
2. 拖动中间的滑块左右移动
3. 左侧显示原图，右侧显示试衣结果

**下载结果**：
- 点击右下角的 **"下载"** 按钮
- 图像会保存到本地

---

### 步骤 6：继续调整（可选）

如果对结果不满意，可以：

1. **修改描述**：在输入框中输入新的服装描述
2. **无需重新上传**：直接点击发送，系统会使用当前画布上的图像
3. **切换区域**：在控制面板中选择不同的替换区域

## 高级技巧

### 1. 提示词优化

**包含详细信息**：
- ✅ 颜色：`深蓝色`、`浅灰色`
- ✅ 材质：`皮革`、`羊绒`、`牛仔布`
- ✅ 风格：`休闲`、`正式`、`运动`
- ✅ 细节：`带拉链`、`有口袋`、`条纹图案`

**保持风格一致**：
```
一件黑色皮夹克，自然光照，与室内办公环境相匹配
```

### 2. 多次尝试

AI 生成具有随机性，如果第一次结果不理想：
- 尝试修改描述的细节
- 多生成几次选择最佳结果
- 调整替换区域（上衣 → 全身）

### 3. 渐进式编辑

对于复杂的换装需求：
1. 先替换上衣
2. 再替换下装
3. 最后调整整体效果

## 常见问题

### Q1: 为什么生成的图像不自然？

**可能原因**：
- 描述与原图风格不匹配
- 替换区域选择不当
- 原图质量较差

**解决方案**：
- 在描述中加入光照信息：`自然光照`、`柔和阴影`
- 尝试不同的替换区域
- 使用更清晰的原图

### Q2: 生成速度很慢？

**正常情况**：
- 首次生成：20-30 秒
- 后续生成：10-20 秒

**如果超过 1 分钟**：
- 检查网络连接
- 刷新页面重试
- 联系管理员检查后端服务

### Q3: 如何获得最佳效果？

**照片要求**：
- 人物正面或侧面清晰可见
- 服装完整，无遮挡
- 光照均匀，对比度适中

**描述要求**：
- 详细具体，包含颜色、材质、风格
- 与原图风格一致
- 避免模糊的词汇（如"好看"、"漂亮"）

### Q4: 可以替换多件服装吗？

**可以**，有两种方式：

**方式 1：分步替换**
1. 先替换上衣
2. 使用生成的结果作为新的原图
3. 再替换下装

**方式 2：全身替换**
1. 选择"全身"模式
2. 在描述中同时描述上衣和下装
```
深蓝色西装外套，白色衬衫，黑色西裤，棕色皮鞋
```

## 技术说明

### 后端 API

系统会优先尝试使用后端 `/api/tryon/edit` API（支持 Vertex AI）：

**成功情况**：
- 使用 Imagen 3 进行高质量编辑
- 支持精确的掩码控制

**失败情况**：
- 自动回退到 Gemini API 智能编辑
- 无需掩码，直接基于描述生成

### 配置要求

**后端配置**（可选）：
```bash
# backend/.env
GCP_PROJECT_ID=your-gcp-project-id
GCP_LOCATION=us-central1
GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json
```

**如果不配置 GCP**：
- 系统会自动使用 Gemini API
- 功能完全可用，只是编辑方式不同
