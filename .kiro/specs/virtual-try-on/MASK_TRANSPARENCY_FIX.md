# 掩码预览透明度和精准度优化报告

## 修复日期
2025-12-20

## 问题描述

用户反馈：
1. **透明度太高**：红色遮罩透明度太高，导致不明显
2. **精准度不足**：掩码覆盖的区域感觉不够精准

## 根本原因

### 问题分析

#### 1. 透明度问题

**原始代码**（第 583-589 行）：
```typescript
const alpha = 0.5;  // 50% 透明度
imageData.data[i] = imageData.data[i] * (1 - alpha) + 255 * alpha;     // R
imageData.data[i + 1] = imageData.data[i + 1] * (1 - alpha);           // G
imageData.data[i + 2] = imageData.data[i + 2] * (1 - alpha);           // B
```

- `alpha = 0.5` 表示 50% 透明度
- 红色叠加后，原图仍然占 50% 的权重
- 导致红色不够明显，用户难以看清标记区域

#### 2. 精准度问题

**原始代码**（第 580 行）：
```typescript
if (brightness > 128) {  // 阈值太高
    // 叠加红色
}
```

- 阈值 `brightness > 128` 太高
- 只有非常亮的像素（接近白色）才会被标记
- Gemini API 返回的掩码可能包含灰度值（0-255），不是纯黑白
- 较暗的掩码区域（如 brightness = 100）被忽略，导致覆盖不完整

---

## 解决方案

### 修改策略

1. **增加透明度**：将 `alpha` 从 0.5 增加到 0.7，使红色更明显
2. **降低阈值**：将阈值从 128 降低到 50，包含更多掩码区域
3. **添加调试日志**：输出掩码统计信息，帮助诊断问题

### 修改的文件

#### `frontend/services/providers/google/media/virtual-tryon.ts` ✅

**修改内容**：

```typescript
// 获取掩码像素数据
const maskData = tempCtx.getImageData(0, 0, img.width, img.height);

// ✅ 添加掩码统计（用于调试）
let maskPixelCount = 0;
let totalPixels = maskData.data.length / 4;
for (let i = 0; i < maskData.data.length; i += 4) {
  const brightness = maskData.data[i];
  if (brightness > 50) {
    maskPixelCount++;
  }
}
console.log(`[generateMaskPreview] 掩码统计: ${maskPixelCount}/${totalPixels} 像素 (${(maskPixelCount/totalPixels*100).toFixed(2)}%)`);

// 在原图上绘制半透明红色
const imageData = ctx.getImageData(0, 0, img.width, img.height);
for (let i = 0; i < maskData.data.length; i += 4) {
  const brightness = maskData.data[i];
  if (brightness > 50) {  // ✅ 降低阈值从 128 到 50
    const alpha = 0.7;  // ✅ 增加透明度从 0.5 到 0.7
    imageData.data[i] = imageData.data[i] * (1 - alpha) + 255 * alpha;     // R
    imageData.data[i + 1] = imageData.data[i + 1] * (1 - alpha);           // G
    imageData.data[i + 2] = imageData.data[i + 2] * (1 - alpha);           // B
  }
}
```

## 视觉效果对比

### 修改前
- **透明度**：50% - 红色较淡，与原图混合明显
- **覆盖范围**：只标记非常亮的区域（brightness > 128）
- **用户体验**：不够明显，难以看清标记区域

### 修改后
- **透明度**：70% - 红色更鲜艳，更容易识别
- **覆盖范围**：包含更多掩码区域（brightness > 50）
- **用户体验**：标记清晰，覆盖更完整

## 阈值选择说明

### 为什么选择 50？

| 阈值 | 覆盖范围 | 优点 | 缺点 |
|------|---------|------|------|
| **> 200** | 极少 | 只标记最亮区域 | 覆盖不完整 ❌ |
| **> 128** | 较少 | 标记中等亮度以上 | 遗漏较暗区域 ❌ |
| **> 50** | 适中 | 包含大部分掩码区域 | 平衡精准度和覆盖 ✅ |
| **> 10** | 很多 | 覆盖几乎所有区域 | 可能包含噪点 ⚠️ |

选择 `> 50` 是一个平衡点：
- 包含 Gemini API 返回的大部分掩码区域
- 过滤掉非常暗的噪点（brightness < 50）
- 提供更完整的覆盖，同时保持精准度

---

## 透明度选择说明

### 为什么选择 0.7？

| Alpha 值 | 红色占比 | 原图占比 | 视觉效果 |
|----------|---------|---------|---------|
| **0.3** | 30% | 70% | 红色太淡 ❌ |
| **0.5** | 50% | 50% | 红色不够明显 ❌ |
| **0.7** | 70% | 30% | 红色明显，原图可见 ✅ |
| **0.9** | 90% | 10% | 红色过于鲜艳，遮挡原图 ⚠️ |

选择 `0.7` 的原因：
- 红色足够明显，用户可以清楚看到标记区域
- 原图仍然可见（30%），用户可以看到服装细节
- 不会过度遮挡原图，保持良好的视觉体验

## 进一步优化建议

如果用户仍然觉得精准度不足，可以考虑：

### 1. 使用 Alpha 通道
```typescript
// 使用掩码的 alpha 通道而不是亮度值
const alpha_channel = maskData.data[i + 3];  // A 通道
if (alpha_channel > 50) {
    // 叠加红色
}
```

### 2. 可调节的透明度
在 UI 中添加滑块，让用户自定义透明度：
```typescript
// 用户可以选择 0.5 - 0.9 之间的透明度
const alpha = userSelectedAlpha;  // 从 UI 获取
```

### 3. 可调节的阈值
在 UI 中添加滑块，让用户自定义阈值：
```typescript
// 用户可以选择 10 - 200 之间的阈值
if (brightness > userSelectedThreshold) {
    // 叠加红色
}
```

### 4. 边缘平滑
使用渐变透明度，使边缘更平滑：
```typescript
// 根据掩码亮度调整透明度
const maskAlpha = brightness / 255;
const finalAlpha = 0.7 * maskAlpha;
```
