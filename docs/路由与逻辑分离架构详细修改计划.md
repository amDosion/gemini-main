## 一、总体目标

实现路由层与业务逻辑完全分离，统一所有 Provider 为协调者模式，使用委托模式内部分发到子服务。

### 核心原则

1. **统一协调者模式**：所有 Provider 都应有协调者服务，统一使用委托模式
2. **路由层简化**：路由层只负责认证、获取凭证、创建服务、调用服务方法
3. **服务内部分发**：协调者服务内部根据 mode 或方法名分发到子服务
4. **向后兼容**：保持现有子服务不变，只添加协调者层

### 重要说明：Google 图片编辑子模式

Google 的图片编辑功能有多个子模式，这些子模式都通过同一个 `edit_image` 方法处理，但需要传递不同的 `mode` 参数：

- **`image-chat-edit`**：对话式图片编辑 → `ConversationalImageEditService`
- **`image-mask-edit`**：遮罩编辑 → `ImageEditCoordinator` (Vertex AI Imagen)
- **`image-inpainting`**：图片修复 → `ImageEditCoordinator` (Vertex AI Imagen)
- **`image-background-edit`**：背景编辑 → `ImageEditCoordinator` (Vertex AI Imagen)
- **`image-recontext`**：图片重上下文 → `ImageEditCoordinator` (Vertex AI Imagen)
- **`image-edit`**：通用编辑（自动检测） → `ImageEditCoordinator` 或 `SimpleImageEditService`

**关键点**：
- 路由层需要将 URL 路径中的 `mode` 参数传递给 `edit_image()` 方法
- `GoogleService.edit_image()` 内部会根据 `mode` 参数智能分发到对应的子服务
- 这是协调者模式的典型应用：路由层调用协调者方法，协调者内部分发到子服务

## 九、Google 图片编辑子模式详细说明

### 9.1 子模式列表

Google 的图片编辑功能支持以下子模式，这些子模式都通过 `/api/modes/google/{mode}` 路由访问，但都映射到 `GoogleService.edit_image()` 方法：

| 子模式 | 路由 | 方法 | mode 参数 | 内部分发目标 |
|--------|------|------|-----------|------------|
| `image-chat-edit` | `/api/modes/google/image-chat-edit` | `edit_image` | `'image-chat-edit'` | `ConversationalImageEditService` |
| `image-mask-edit` | `/api/modes/google/image-mask-edit` | `edit_image` | `'image-mask-edit'` | `ImageEditCoordinator` (Vertex AI Imagen) |
| `image-inpainting` | `/api/modes/google/image-inpainting` | `edit_image` | `'image-inpainting'` | `ImageEditCoordinator` (Vertex AI Imagen) |
| `image-background-edit` | `/api/modes/google/image-background-edit` | `edit_image` | `'image-background-edit'` | `ImageEditCoordinator` (Vertex AI Imagen) |
| `image-recontext` | `/api/modes/google/image-recontext` | `edit_image` | `'image-recontext'` | `ImageEditCoordinator` (Vertex AI Imagen) |
| `image-edit` | `/api/modes/google/image-edit` | `edit_image` | `None` 或自动检测 | `ImageEditCoordinator` 或 `SimpleImageEditService` |

### 9.2 路由层处理逻辑

**在 `modes.py` 路由中的处理**：

```python
# ✅ 5. 准备参数
method = getattr(service, method_name)

# 构建调用参数
params = {
    "model": request_body.modelId,
    "prompt": request_body.prompt,
    **(request_body.options or {}),
    **(request_body.extra or {})
}

# **重要**：对于 edit_image 方法，需要传递 mode 参数
# GoogleService.edit_image() 会根据 mode 参数智能分发到不同的子服务
if method_name == "edit_image":
    # 将 URL 路径中的 mode 参数传递给 edit_image 方法
    params["mode"] = mode

# 处理 reference_images 参数（从 attachments 中提取）
# reference_images 字典格式：{'raw': base64_image, 'mask': base64_mask, ...}
if request_body.attachments:
    reference_images = {}
    for attachment in request_body.attachments:
        if attachment.get('type') == 'image':
            if attachment.get('role') == 'mask':
                reference_images['mask'] = attachment.get('url') or attachment.get('base64Data')
            else:
                reference_images['raw'] = attachment.get('url') or attachment.get('base64Data')
    if reference_images:
        params["reference_images"] = reference_images

# ✅ 6. 调用服务方法（服务内部会分发到子服务）
result = await method(**params)
```

### 9.3 GoogleService.edit_image() 内部分发逻辑

`GoogleService.edit_image()` 方法内部已经实现了智能分发逻辑（按优先级）：

1. **`mode='image-chat-edit'`** → 使用 `ConversationalImageEditService`（对话式编辑）
2. **`mode='image-mask-edit'`** → 使用 `ImageEditCoordinator` (Vertex AI Imagen，精确编辑，必须有 mask)
3. **`mode='image-inpainting'`** → 使用 `ImageEditCoordinator` (Vertex AI Imagen，图片修复)
4. **`mode='image-background-edit'`** → 使用 `ImageEditCoordinator` (Vertex AI Imagen，背景编辑)
5. **`mode='image-recontext'`** → 使用 `ImageEditCoordinator` (Vertex AI Imagen，重新上下文)
6. **有 mask** → 使用 `ImageEditCoordinator` (Vertex AI Imagen，精确编辑，自动检测)
7. **没有 mask** → 使用 `SimpleImageEditService` (generateContent 方式，简单编辑)
