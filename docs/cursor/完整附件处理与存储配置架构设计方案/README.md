## 📖 文档内容概览

### requirements.md - 需求文档

**主要内容**:
- 项目背景和现状问题
- 项目目标和成功标准
- 功能需求（FR-001 至 FR-005）
- 非功能需求（性能、可靠性、安全性、可维护性、可扩展性）
- 约束和假设
- 验收标准
- 风险和假设

**核心需求**:
1. **FR-001**: 修复 `storage.py` 中的配置解密
2. **FR-002**: 统一配置管理（可选，长期优化）
3. **FR-003**: 修复URL类型判断
4. **FR-004**: 确保跨模式附件显示
5. **FR-005**: 合并加密文件到 `core/encryption.py`

---

### design.md - 设计文档

**主要内容**:
- 系统架构设计（架构原则、架构分层）
- 后端统一工具架构（加密/解密工具、请求头处理）
- 前端图片显示与跨模式流程（AI生成图片流程、跨模式流程、URL类型判断）
- 存储配置管理架构（配置处理流程、统一配置获取方法）
- 加密文件合并设计（合并方案、合并后的结构、关键修改点）
- 数据流设计（AI生成图片完整流程、跨模式切换完整流程）
- API设计（存储配置API、附件API）
- 关键技术方案（配置解密方案、前端URL类型判断方案、加密文件合并方案）
- 安全性设计（密钥管理、敏感数据保护、认证和授权）
- 监控和日志设计（日志记录、监控指标）

**核心设计**:
1. **统一配置管理**: 优先使用 `StorageManager`，或使用 `decrypt_config()`
2. **附件显示与上传状态解耦**: 显示逻辑只依赖URL有效性
3. **统一认证处理**: 使用 `Depends(require_current_user)`
4. **统一加密管理**: 所有加密功能统一在 `core/encryption.py`

### tasks.md - 任务文档

**主要内容**:
- 任务总览（里程碑、优先级定义）
- 阶段1: 修复后端配置解密问题（TASK-101 至 TASK-103）
- 阶段2: 优化前端跨模式附件显示（TASK-201 至 TASK-205）
- 阶段3: 合并加密文件（TASK-301 至 TASK-306）
- 阶段4: 统一配置管理（可选，TASK-401 至 TASK-403）
- 任务依赖关系（依赖图、关键路径）
- 风险缓解任务（TASK-701 至 TASK-703）
- 任务时间估算

**核心任务**:
- **P0 任务**: 5个（关键，必须完成）
- **P1 任务**: 12个（重要）
- **P2 任务**: 3个（次要，可选）

**总工时估算**:
- 必须完成的任务: 21.5小时（约3天）
- 可选任务: 5小时（约0.5天）
- **总计**: 26.5小时（约3.5天）

## 🎯 使用指南

### 对于项目经理

1. **阅读 requirements.md**: 了解项目背景、问题和需求
2. **阅读 tasks.md**: 了解任务分解、时间估算和依赖关系
3. **跟踪进度**: 使用 tasks.md 中的里程碑和任务列表跟踪项目进度

### 对于开发人员

1. **阅读 requirements.md**: 了解功能需求和非功能需求
2. **阅读 design.md**: 了解系统架构和技术方案
3. **阅读 tasks.md**: 了解具体任务和实施步骤
4. **阅读 TODOS.md**: 了解详细的任务列表和实施计划
5. **阅读 CODE_REVIEW_CHECKLIST.md**: 在提交代码前进行自检
6. **按照 TODOS.md 执行**: 按照待办事项列表中的详细步骤实施

### 对于架构师

1. **阅读 design.md**: 了解完整的架构设计和技术方案
2. **审查 requirements.md**: 确保设计满足所有需求
3. **审查 tasks.md**: 确保任务分解合理，依赖关系清晰
4. **审查 CODE_REVIEW_CHECKLIST.md**: 确保代码审查检查清单完整

### 对于代码审查者

1. **阅读 CODE_REVIEW_CHECKLIST.md**: 了解代码审查检查清单
2. **使用检查清单**: 在代码审查时使用检查清单逐项检查
3. **记录问题**: 对于不符合要求的代码，记录具体问题
