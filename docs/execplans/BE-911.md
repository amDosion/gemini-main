# BE-911

## Goal

Move remaining standalone video business semantics behind the backend contract so frontend can stop deriving or enforcing policy locally.

## Scope

- Keep scope limited to standalone video mode.
- Do not refactor workflow video execution in this step.
- Focus on backend request normalization and schema contract semantics.

## Changes

- Extended `backend/app/services/common/video_mode_contract.py`
  - added field policies for enhance prompt, subtitle mode, runtime-only options, and storyboard deprecation hints
  - added backend request normalization for `video-gen`
  - added model-aware extension support checks
- Updated `backend/app/services/gemini/base/video_storyboard.py`
  - explicit storyboard prompt now suppresses legacy tracking companion fields
- Updated `backend/app/routers/core/modes.py`
  - `generate_video` requests now run through backend request normalization after catalog validation
