# BE-910

## Goal

Unify standalone video-mode contract semantics in backend code so frontend can consume a single backend-defined schema instead of deriving business rules locally.

## Scope

- Keep work limited to standalone video mode.
- Do not change workflow video behavior in this step.
- Move router-resident video attachment semantics into backend helpers.
- Expose a backend-derived video contract on controls schema responses.

## Changes

- Added `backend/app/services/common/video_mode_contract.py`
  - runtime-aware controls schema resolution
  - Google video runtime overrides
  - attachment-slot normalization for generate / understand / delete video flows
  - backend-derived `video_contract` payload
- Refactored `backend/app/routers/core/modes.py`
  - controls endpoint now uses backend runtime contract helper
  - video attachment parameter merging now uses backend helper instead of router-local business logic
