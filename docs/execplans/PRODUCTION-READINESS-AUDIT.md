## CRITICAL 问题（必须修复）

### 1. 安全：存储端点完全无认证
- **文件**: `routers/storage/storage.py`
- 上传、列表、获取、删除文件均无任何认证
- **风险**: 任何人可上传恶意文件、窃取数据、耗尽存储
- **修复**: 添加 API Key / JWT 认证中间件

### 2. 安全：Browse 工具存在 SSRF 漏洞
- **文件**: `routers/tools/browse.py`
- 用户提供的 URL 直接请求，无内网 IP 过滤
- **风险**: 攻击者可探测内网、获取云元数据（GCP credentials）
- **修复**: 校验 URL scheme，封锁 RFC 1918/link-local/169.254.169.254

### 3. 安全：Browse 工具无认证
- **文件**: `routers/tools/browse.py`
- 任何人可通过此端点代理请求任意 URL
- **修复**: 添加认证，限制仅已登录用户可用

### 4. 性能：同步数据库阻塞事件循环
- **文件**: `core/database.py`
- 使用 `create_engine()`（同步），所有 DB 查询阻塞 asyncio
- **影响**: 高并发下整个服务器被 DB 查询阻塞
- **修复**: 迁移到 `create_async_engine()` 或用 `asyncio.to_thread()` 包装

### 5. 数据库：所有 user_id 列缺少索引
- **涉及**: chat_sessions、user_settings、vertex_ai_configs、file_records、interactions
- **影响**: 随数据增长，所有按用户查询变成全表扫描
- **修复**: 为所有 `user_id` 列添加数据库索引

### 6. 数据库：凭证加密密钥管理不明
- **文件**: `models/vertex_config.py`
- Vertex AI credentials 加密存储，但密钥来源和轮转策略不清晰
- **修复**: 确保加密密钥来自安全源（KMS/Vault），文档化轮转流程

### 7. 部署：完全没有 CI/CD 流水线
- 无 GitHub Actions、GitLab CI、Jenkins 或任何 CI 配置
- **影响**: 无自动化测试、lint、安全扫描、部署
- **修复**: 建立 CI 流水线（lint + test + security scan + build）

### 8. 依赖：所有 46 个 Python 包版本未锁定
- `requirements.txt` 全部使用 `>=` 无上限约束
- **影响**: 不同时间安装可能得到不兼容版本
- **修复**: 锁定精确版本，生成 lock 文件

### 9. 韧性：无任何外部服务的熔断器
- 所有 AI API 调用无 circuit breaker
- **影响**: 上游服务宕机时所有请求挂起，级联故障
- **修复**: 为每个 Provider 实现熔断器模式

### 10. 测试：前端零测试覆盖
- 无 unit test、component test、E2E test
- 连测试框架都未配置
- **影响**: 任何 UI 变更均无法自动验证
- **修复**: 配置 Vitest + 编写关键路径测试

## 结论

当前产品的**核心功能完整**，但在安全性、可靠性和运维方面存在显著差距。建议：

1. **P0 项（~4 天）为上线硬性门槛**，必须完成才能对外服务
2. **P1 项（~7 天）在上线后第一周集中解决**，确保生产环境稳定
3. **P2/P3 项纳入后续迭代**，持续改进

最大的风险点是：**安全漏洞**（SSRF、无认证端点）和**性能瓶颈**（同步数据库阻塞事件循环）。这两项如不修复，上线后极可能出现安全事件或服务不可用。
