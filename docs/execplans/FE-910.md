# FE-910

## Goal

Align standalone video-mode frontend controls to the backend-defined `video_contract` so the page stops deriving extension, subtitle, and prompt-enhancement policy locally.

## Changes

- Extended [`frontend/hooks/useModeControlsSchema.ts`](frontend/hooks/useModeControlsSchema.ts) to parse backend `video_contract`, including:
  - `field_policies`
  - `extension_duration_matrix`
  - `extension_constraints`
  - `attachment_slots`
  - `input_strategies`
- Updated [`frontend/utils/videoControlSchema.ts`](frontend/utils/videoControlSchema.ts) so `buildVideoControlContract()` now prefers backend-derived:
  - extension counts by base duration
  - subtitle default-enabled mode
  - enhance-prompt mandatory/locked semantics
  - extension constraints metadata
- Updated [`frontend/controls/modes/google/VideoGenControls.tsx`](frontend/controls/modes/google/VideoGenControls.tsx) to stop locally inventing:
  - extension duration options
  - subtitle enable mode
  - enhance-prompt mandatory policy
  - storyboard prompting availability
