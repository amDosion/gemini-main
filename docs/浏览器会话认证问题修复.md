# 浏览器会话认证问题修复

## 问题描述

后端日志出现警告：
```
2026-01-19 19:50:54 - browser - WARNING - [WARN] No browser session found for user: default
```

## 问题原因

1. **`/api/browse` 端点未要求认证**
   - 端点没有使用 `Depends(require_current_user)` 来获取 user_id
   - 调用 `selenium_browse()` 时没有传递 `user_id` 参数
   - 导致使用默认值 "default"，但该用户可能没有浏览器会话

2. **`cleanup()` 函数使用错误的清理方法**
   - `cleanup()` 函数调用 `close_driver()` 时没有传入 user_id
   - 使用默认值 "default"，如果该用户没有会话就会输出警告

## 修复方案

### 1. 修复 `/api/browse` 端点

**文件**: `backend/app/routers/tools/browse.py`

**修改内容**:
- ✅ 添加 `user_id: str = Depends(require_current_user)` 参数
- ✅ 调用 `_selenium_browse()` 时传递 `user_id` 参数
- ✅ 在日志中记录 user_id

**修改前**:
```python
@router.post("/browse", response_model=BrowseResponse)
async def browse_webpage(request: BrowseRequest):
    # ...
    content = _selenium_browse(url, steps=[
        {"action": "wait", "seconds": 2}
    ])
```

**修改后**:
```python
@router.post("/browse", response_model=BrowseResponse)
async def browse_webpage(
    request: BrowseRequest,
    user_id: str = Depends(require_current_user)  # ✅ 添加认证
):
    # ...
    # ✅ 传递 user_id 以实现会话隔离
    content = _selenium_browse(url, steps=[
        {"action": "wait", "seconds": 2}
    ], user_id=user_id)
```

### 2. 修复 `cleanup()` 函数

**文件**: `backend/app/services/gemini/browser.py`

**修改内容**:
- ✅ 使用 `close_all_drivers()` 而不是 `close_driver()`
- ✅ 避免警告（因为会关闭所有用户的会话，而不是只关闭 "default" 用户的会话）

**修改前**:
```python
def cleanup():
    """
    Clean up resources (close browser, etc.)
    """
    close_driver()  # ❌ 使用默认值 "default"，可能没有会话
```

**修改后**:
```python
def cleanup():
    """
    Clean up resources (close browser, etc.)
    
    注意：此函数在模块卸载时调用，会关闭所有用户的浏览器会话
    """
    close_all_drivers()  # ✅ 关闭所有用户的会话，避免警告
```

## 影响范围

### 修复前
- ❌ `/api/browse` 端点不需要认证，任何人都可以调用
- ❌ 浏览器会话使用 "default" 用户，无法实现多用户隔离
- ❌ 模块卸载时可能输出警告

### 修复后
- ✅ `/api/browse` 端点要求认证，必须提供有效的 token
- ✅ 浏览器会话按用户隔离，每个用户有独立的会话
- ✅ 模块卸载时正确清理所有会话，不会输出警告

## 验证方法

修复后，应该：
1. ✅ `/api/browse` 端点要求认证（未认证会返回 401）
2. ✅ 调用 `/api/browse` 时，日志显示正确的 user_id（不是 "default"）
3. ✅ 不再出现 "No browser session found for user: default" 警告
4. ✅ 每个用户有独立的浏览器会话
