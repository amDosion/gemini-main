# 新用户首次使用流程

## 流程概述

当新用户首次登录时，由于没有配置任何 AI 提供商，系统会引导用户完成初始配置。

## 完整流程

### 1. 用户登录

- 用户通过登录页面登录系统
- 系统验证用户身份，获取 access_token 和 refresh_token

### 2. 加载初始化数据

- 前端调用 `/api/init` API 获取用户初始化数据
- 后端返回：
  - `profiles`: 配置列表（新用户为空数组 `[]`）
  - `activeProfileId`: 激活的配置 ID（新用户为 `null`）
  - `activeProfile`: 激活的配置对象（新用户为 `null`）
  - `sessions`: 会话列表（新用户为空数组 `[]`）
  - `personas`: 角色列表（自动初始化默认角色）
  - 其他配置数据

### 3. 显示欢迎界面

**判断条件**：
```typescript
if (isConfigReady && !activeProfileFromSettings) {
  return <WelcomeScreen onOpenSettings={() => handleOpenSettings('editor')} />;
}
```

- 当 `isConfigReady` 为 `true`（配置已加载完成）
- 且 `activeProfileFromSettings` 为 `null`（没有激活的配置）
- 系统显示欢迎界面

**欢迎界面内容**：
- 标题：欢迎使用！
- 消息：请先配置您的 AI 提供商以开始使用
- 按钮：打开设置

### 4. 用户点击"打开设置"

- 调用 `handleOpenSettings('editor')`
- 设置 `settingsInitialTab` 为 `'editor'`
- 打开设置模态框，**直接进入编辑器标签页**

### 5. 创建第一个配置

在编辑器标签页中：

1. **选择提供商**：从下拉列表中选择一个 AI 提供商（如 Google Gemini、OpenAI、DeepSeek 等）
2. **填写配置信息**：
   - API Key（必需）
   - Base URL（可选，有默认值）
   - 配置名称（可选，有默认值）
3. **验证配置**（可选）：
   - 点击"验证"按钮测试 API Key 是否有效
   - 系统会获取可用的模型列表
4. **保存配置**：
   - 点击"保存"按钮
   - 系统会：
     - 保存配置到后端
     - 自动激活该配置（`autoActivate: true`）
     - 刷新设置状态
     - 关闭设置模态框

### 6. 配置完成

- `useSettings` hook 会调用 `refreshSettings()` 更新状态
- `activeProfileFromSettings` 从 `null` 变为配置对象
- 欢迎界面自动消失
- 显示主应用界面

## 优化点

### 已实现的优化

1. **直接进入编辑器**：
   - 从欢迎界面点击"打开设置"时，直接进入编辑器标签页，而不是配置列表
   - 减少用户操作步骤

2. **空状态优化**：
   - 在配置列表为空时，显示明显的"创建第一个配置"按钮
   - 提供更好的视觉引导

3. **自动激活**：
   - 创建第一个配置时自动激活，无需额外操作

4. **状态同步**：
   - 使用 `activeProfileFromSettings` 而不是 `initData?.activeProfile` 判断
   - 确保保存配置后立即更新界面状态

### 可能的进一步优化

1. **引导提示**：
   - 在编辑器标签页添加引导提示，说明如何获取 API Key

2. **快速开始模板**：
   - 提供常用提供商的快速配置模板

3. **配置验证提示**：
   - 在保存前提示用户验证配置

## 故障排除

### 问题：保存配置后仍然显示欢迎界面

**原因**：
- `activeProfileFromSettings` 没有正确更新

**解决方案**：
1. 检查浏览器控制台是否有错误
2. 检查网络请求，确认配置是否成功保存
3. 手动刷新页面

### 问题：无法打开设置模态框

**原因**：
- 可能是状态管理问题

**解决方案**：
1. 检查 `isSettingsOpen` 状态
2. 检查浏览器控制台错误
3. 尝试手动调用 `setIsSettingsOpen(true)`

### 问题：创建配置后没有自动激活

**原因**：
- `autoActivate` 参数可能没有正确传递

**解决方案**：
1. 检查 `EditorTab` 的 `handleSave` 函数
2. 确认 `onSaveProfile(profile, true)` 的第二个参数为 `true`
