# 存储配置加密警告问题修复

## 问题描述

后端日志出现警告：
```
2026-01-19 19:56:52 - backend.app.utils.encryption - WARNING - Field 'accessKeyId' appears to be unencrypted or encrypted with a different key. Using value as-is.
2026-01-19 19:56:52 - backend.app.utils.encryption - WARNING - Field 'accessKeySecret' appears to be unencrypted or encrypted with a different key. Using value as-is.
```

## 问题原因

### 1. 历史数据未加密
数据库中可能有一些旧的存储配置，这些配置在添加加密功能之前就已经保存了，所以 `accessKeyId` 和 `accessKeySecret` 字段是明文的。

### 2. 解密逻辑问题
`decrypt_config()` 函数在尝试解密明文字段时，会抛出 `InvalidToken` 异常，然后输出警告。虽然最终会使用原值（明文），但警告信息会误导用户。

### 3. 加密逻辑问题
`encrypt_config()` 函数没有检查字段是否已经加密，可能导致重复加密（虽然 Fernet 加密是幂等的，但检查可以避免不必要的操作）。

## 修复方案

### 1. 改进 `encrypt_config()` 函数

**文件**: `backend/app/utils/encryption.py`

**修改内容**:
- ✅ 在加密前检查字段是否已经加密
- ✅ 如果已加密，直接使用，避免重复加密

**修改前**:
```python
elif field in SENSITIVE_FIELDS and isinstance(value, str):
    # Encrypt sensitive string fields
    try:
        encrypted_bytes = fernet.encrypt(value.encode('utf-8'))
        encrypted_config[field] = encrypted_bytes.decode('utf-8')
    except Exception as e:
        logger.error(f"Failed to encrypt field '{field}': {e}")
        encrypted_config[field] = value
```

**修改后**:
```python
elif field in SENSITIVE_FIELDS and isinstance(value, str):
    # Encrypt sensitive string fields
    # ✅ 检查是否已经加密，避免重复加密
    if is_encrypted(value):
        # 已经加密，直接使用
        encrypted_config[field] = value
    else:
        # 未加密，进行加密
        try:
            encrypted_bytes = fernet.encrypt(value.encode('utf-8'))
            encrypted_config[field] = encrypted_bytes.decode('utf-8')
        except Exception as e:
            logger.error(f"Failed to encrypt field '{field}': {e}")
            encrypted_config[field] = value
```

### 2. 改进 `decrypt_config()` 函数

**文件**: `backend/app/utils/encryption.py`

**修改内容**:
- ✅ 在解密前检查字段是否已加密
- ✅ 如果未加密（历史数据），直接使用，不输出警告
- ✅ 只在真正解密失败时输出警告

**修改前**:
```python
elif field in SENSITIVE_FIELDS and isinstance(value, str):
    # Decrypt sensitive string fields
    try:
        decrypted_bytes = fernet.decrypt(value.encode('utf-8'))
        decrypted_config[field] = decrypted_bytes.decode('utf-8')
    except InvalidToken:
        logger.warning(
            f"Field '{field}' appears to be unencrypted or encrypted with a different key. "
            "Using value as-is."
        )
        # If decryption fails, assume it's already decrypted
        decrypted_config[field] = value
```

**修改后**:
```python
elif field in SENSITIVE_FIELDS and isinstance(value, str):
    # Decrypt sensitive string fields
    # ✅ 检查是否已加密，避免对明文进行解密尝试
    if is_encrypted(value):
        # 已加密，尝试解密
        try:
            decrypted_bytes = fernet.decrypt(value.encode('utf-8'))
            decrypted_config[field] = decrypted_bytes.decode('utf-8')
        except InvalidToken:
            logger.warning(
                f"Field '{field}' appears to be encrypted with a different key. "
                "Using value as-is."
            )
            # 如果解密失败（可能是不同的密钥），保持原值
            decrypted_config[field] = value
        except Exception as e:
            logger.error(f"Failed to decrypt field '{field}': {e}")
            decrypted_config[field] = value
    else:
        # 未加密（可能是历史数据），直接使用
        logger.debug(
            f"Field '{field}' is not encrypted (likely historical data). Using value as-is."
        )
        decrypted_config[field] = value
```

## 修复效果

### 修复前
- ❌ 每次解密历史数据时都会输出警告
- ❌ 警告信息可能误导用户（以为是加密密钥问题）
- ❌ 可能重复加密已加密的字段

### 修复后
- ✅ 历史数据（明文）直接使用，不输出警告
- ✅ 只在真正解密失败时输出警告（如密钥不匹配）
- ✅ 避免重复加密已加密的字段
- ✅ 使用 `logger.debug()` 记录历史数据的使用（不会在生产环境输出）

## 数据迁移建议

如果希望将所有历史数据加密，可以：

1. **手动重新保存配置**：在设置页面重新保存存储配置，后端会自动加密
2. **编写迁移脚本**：批量读取所有存储配置，重新加密并保存

**注意**：历史数据（明文）仍然可以正常使用，只是没有加密保护。如果安全要求高，建议重新保存配置以启用加密。

## 验证方法

修复后，应该：
1. ✅ 不再出现 "Field 'accessKeyId' appears to be unencrypted" 警告（对于历史数据）
2. ✅ 历史数据（明文）可以正常使用
3. ✅ 新保存的配置会自动加密
4. ✅ 已加密的配置可以正常解密和使用
