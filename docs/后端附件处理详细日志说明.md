## 日志格式

### 日志级别

- **INFO**：正常流程和成功操作
- **WARNING**：警告信息（如数据不一致）
- **ERROR**：错误信息（如验证失败、异常）

### 日志标识

- `==========`：流程开始/结束标记
- `📥`：请求参数
- `📋`：任务参数
- `🔍`：查找/检查操作
- `🔄`：处理中
- `✅`：成功
- `❌`：失败
- `⚠️`：警告
- `⏭️`：跳过

### 耗时统计

所有关键步骤都包含耗时统计（毫秒）：
- 每个步骤的耗时
- 总耗时

## 使用场景

### 排查多余上传

**查看日志**：
1. 查找 `[AttachmentService] ✅ [步骤3] 附件已上传完成，直接复用`
2. 如果看到这个日志，说明已跳过上传任务创建
3. 如果没有看到，检查是否创建了新的上传任务

**关键日志**：
```
[AttachmentService] ✅ [步骤3] 附件已上传完成，直接复用
[AttachmentService]     - 跳过上传任务创建
```

### 排查响应慢

**查看日志**：
1. 查看每个步骤的耗时统计
2. 找出耗时最长的步骤
3. 分析该步骤的性能瓶颈

**关键日志**：
```
[AttachmentService] ✅ [步骤1] ... (耗时: 5.12ms)
[AttachmentService] ✅ [步骤2] ... (耗时: 3.45ms)
[AttachmentService] ========== ... (总耗时: 8.57ms) ==========
```

### 排查数据不一致

**查看日志**：
1. 查找 `[AttachmentService] ⚠️ [步骤3] 附件有 HTTP URL 但状态不是 completed`
2. 检查状态更新是否成功

**关键日志**：
```
[AttachmentService] ⚠️ [步骤3] 附件有 HTTP URL 但状态不是 completed，更新状态并直接返回
[AttachmentService]     - 原状态: pending
[AttachmentService]     - 更新为: completed
```
