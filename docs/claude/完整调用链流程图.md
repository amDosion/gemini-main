## 流程一：image-gen 模式（图片生成）

```
1.  用户点击生成按钮
    ↓
2.  llmService.generateImage(modelId, prompt, referenceImages, options, apiKey, baseUrl)
    → 合并 _cachedOptions + 传入 options
    ↓
3.  UnifiedProviderClient.generateImage()                [UnifiedProviderClient.ts:508]
    → 委托给 this.executeMode('image-gen', modelId, prompt, referenceImages, options)
    ↓
4.  UnifiedProviderClient.executeMode()                  [UnifiedProviderClient.ts:380]
    → requestBody = { modelId, prompt, attachments, options, extra }
    → token = localStorage.getItem('access_token')           ← 取出 JWT
    → headers = {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}`                    ← 附上 JWT
      }
    → url = `/api/modes/google/image-gen`
    → fetch(url, { method:'POST', headers, credentials:'include', body: JSON.stringify(requestBody) })
    ↓
5.  FastAPI 路由匹配: POST /api/modes/{provider}/{mode}  [modes.py:181]
    → handle_mode(provider="google", mode="image-gen", request_body, request,
                  user_id=Depends(require_current_user), db=Depends(get_db))
    ↓
6.  require_current_user(request)                        [dependencies.py:18]
    → require_user_id(request)                           [user_context.py:82]
    → get_current_user_id(request)                       [user_context.py:21]
      → auth_header = request.headers.get("Authorization")   ← "Bearer eyJ..."
      → parts = auth_header.split() → ["Bearer", "eyJ..."]
      → token = parts[1]
      → payload = decode_token(token)                    [jwt_utils.py]
        → jwt.decode(token, SECRET_KEY, algorithms=["HS256"])
        → 验证 payload.type == "access"
      → user_id = payload.sub                                ← 提取用户ID
      → 返回 user_id
    ※ token 无效 → 401 Unauthorized
    ↓
7.  get_db()                                             [database.py]
    → yield SessionLocal()                                   ← 注入数据库会话
    ↓
8.  [步骤1] get_provider_credentials(provider="google", db, user_id)
                                                         [credential_manager.py:22]
    → 优先级1: request_api_key 有值? → 直接返回（测试用）
    → 优先级2: 查数据库
      → settings = db.query(UserSettings).filter(user_id=user_id).first()
      → active_profile_id = settings.active_profile_id
      → active_profile = db.query(ConfigProfile).filter(
            id=active_profile_id, provider_id="google", user_id=user_id
        ).first()
      → 解密 API Key:
        → is_encrypted(active_profile.api_key)           [encryption.py:293]
          → 尝试 Fernet 解密来判断是否加密
        → decrypt_data(api_key, silent=True)             [encryption.py:255]
          → key = _get_encryption_key_bytes()     ← 从 ENCRYPTION_KEY 环境变量
          → fernet = Fernet(key)
          → encrypted_bytes = base64.b64decode(api_key.encode())
          → decrypted = fernet.decrypt(encrypted_bytes).decode()
        → 返回 (明文api_key, active_profile.base_url)
    → 优先级3: 回退查任意匹配 provider="google" 的 ConfigProfile
    → 都找不到 → 401 "API Key not found"
    ↓
9.  [步骤2] ProviderFactory.create("google", api_key, api_url, user_id, db)
                                                         [provider_factory.py:58]
    → cache_key = "google:{api_key}"
    → 缓存命中? → 返回 cached GoogleService
    → 缓存未命中:
      → GoogleService(api_key, api_url, user_id=user_id, db=db)
                                                         [google_service.py:71]
        → self.sdk_initializer = SDKInitializer(api_key, api_url)
        → self.image_generator = ImageGenerator(api_key, user_id, db)
                                                         [google_service.py:203]
          → self._coordinator = ImagenCoordinator(user_id, db, api_key)
                                                         [image_generator.py:52]
        → self.image_edit_coordinator = ImageEditCoordinator(user_id, db)
                                                         [google_service.py:217]
          → self._config = _load_config()
            → 从 VertexAIConfig DB 加载（独立于 credential_manager）
      → 缓存 _client_cache[cache_key] = service
      → 返回 GoogleService 实例
    ↓
10. [步骤3] get_service_method("image-gen") → "generate_image"
                                                         [mode_method_mapper.py:15]
    ↓
11. [步骤4] hasattr(service, "generate_image") → True
    ↓
12. [步骤5] 参数构建                                     [modes.py:290-499]
    → params = { "model": request_body.modelId, "prompt": request_body.prompt }
    → params.update(options_dict)    ← numberOfImages, aspectRatio, imageStyle 等
    → 处理 attachments（如果有）:
      → convert_attachments_to_reference_images(attachments)
                                                         [modes.py:120]
      → 查 DB MessageAttachment（如果有 attachment_id）
      → params["reference_images"] = reference_images
    ↓
13. [步骤6] await service.generate_image(**params)
    → GoogleService.generate_image(prompt, model, **kwargs)
                                                         [google_service.py:388]
    ↓
14. self.image_generator.generate_image(prompt, model, **kwargs)
                                                         [image_generator.py:56]
    → _convert_parameters_for_api(kwargs)
      PARAM_MAPPING: numberOfImages→number_of_images, aspectRatio→aspect_ratio ...
    ↓
15. generator = self._coordinator.get_generator()        [imagen_coordinator.py:78]
    → _load_config()                                     [imagen_coordinator.py:153]
      ← 第二条独立配置加载路径
      → db.query(VertexAIConfig).filter(user_id=user_id)
      → 解密 vertex_ai_credentials_json（Fernet）
      → 确定 api_mode: 'vertex_ai' 或 'gemini_api'
    → api_mode == 'vertex_ai'
      → VertexAIImageGenerator(project_id, location, credentials_json)
    → api_mode == 'gemini_api'
      → GeminiAPIImageGenerator(api_key)
    ↓
16. generator.generate_image(prompt, model, **kwargs)
    → 调用 Google GenAI SDK (client.models.generate_images / generate_content)
    → 返回 List[Dict]:
      [{ "url": "data:image/png;base64,...", "mimeType": "image/png", ... }]
    ↓
17. [步骤7] 处理返回结果                                 [modes.py:561-668]
    → attachment_service = AttachmentService(db)
    → session_id = options.frontend_session_id / sessionId
    → message_id = options.message_id
    → 对每张图片:
      → AttachmentService.process_ai_result(
            ai_url, mime_type, session_id, message_id, user_id, prefix="generated"
        )
        → 创建 MessageAttachment DB 记录
        → 启动异步上传任务 (background task)
        → 返回 { attachment_id, display_url, status, task_id }
      → 构建 image_result =
        { url, attachmentId, uploadStatus, taskId, mimeType, filename }
    → result = processed_images
    ↓
18. [步骤8] 返回 ModeResponse                           [modes.py:676]
    → { success: true, data: processed_images, provider: "google", mode: "image-gen" }
    ↓
19. 前端 executeMode() 接收响应                          [UnifiedProviderClient.ts:470]
    → data = await response.json()
    → data.data.images → 映射为 ImageGenerationResult[]
      → { url, mimeType, filename, attachmentId, uploadStatus, taskId, enhancedPrompt }
    ↓
20. 前端渲染图片
```

## ImageEditCoordinator 路由分发（连续判断链）

```
GoogleService.edit_image(mode=?)                         [google_service.py:467]
  ↓
ImageEditCoordinator.edit_image(mode=?)                  [image_edit_coordinator.py:367]
  │
  │ 构造时已执行: _load_config()                          [L136-269]
  │   → db.query(VertexAIConfig).filter(user_id)
  │   → decrypt credentials_json
  │   → 确定 api_mode: 'vertex_ai' | 'gemini_api'
  │
  ├─ if mode=='image-chat-edit'                          [L411]
  │     → ConversationalImageEditService（Gemini API, generateContent）
  │     不需要 Vertex AI
  │
  ├─ elif mode=='image-mask-edit'                        [L436]
  │     → 要求 api_mode=='vertex_ai'
  │     → MaskEditService (mask_edit_service.py)
  │     → 传递 project_id, location, credentials_json
  │     → has_mask? → edit_with_mask() : edit_without_mask()
  │
  ├─ elif mode in ['image-inpainting',
  │                 'image-background-edit',
  │                 'image-recontext']                    [L534]
  │     → 要求 api_mode=='vertex_ai'
  │     → VertexAIImageEditor（通用编辑器）
  │     → edit_mode 映射: inpainting / background_edit / recontext
  │
  ├─ elif 有 mask（自动检测）                              [L578]
  │     → 要求 api_mode=='vertex_ai'
  │     → VertexAIImageEditor
  │
  └─ else 无 mask（自动检测）                              [L599]
        → SimpleImageEditService (deprecated, generateContent)
        不需要 Vertex AI
```

## 两条独立配置加载路径

```
credential_manager [modes.py:242]          ImageEditCoordinator._load_config() [L136]
        │                                            │
        ↓                                            ↓
  UserSettings.active_profile_id             VertexAIConfig (user_id)
        ↓                                            ↓
  ConfigProfile (provider_id="google")       api_mode, project_id, location
        ↓                                            ↓
  decrypt_data(api_key)                      decrypt_data(credentials_json)
        ↓                                            ↓
  用途: ProviderFactory 创建 GoogleService   用途: 路由到 Vertex AI 或 Gemini API
  (api_key 作为缓存键 + Gemini API 认证)     (Vertex AI 服务账号认证)
        │                                            │
        └──── 互不依赖，各自独立加载 ────────────────┘
```

| 维度 | credential_manager | ImageEditCoordinator / ImagenCoordinator |
|------|-------------------|------------------------------------------|
| 触发位置 | modes.py:242 | image_edit_coordinator.py:77 |
| 查询的表 | UserSettings → ConfigProfile | VertexAIConfig |
| 获取的内容 | api_key + base_url | api_mode + project_id + location + credentials_json |
| 解密方法 | decrypt_data(api_key, silent=True) | decrypt_data(credentials_json) |
| 用途 | 创建 GoogleService (ProviderFactory 缓存键) | 路由到 Vertex AI 或 Gemini API |
| 是否互相依赖 | 否 | 否（独立加载） |

> `credential_manager` 返回的 `base_url`（如 `https://generativelanguage.googleapis.com/v1beta`）
> 仅用于 ProviderFactory 缓存键和 Gemini API 聊天，不影响图片编辑路由。
> 图片编辑用的是 `ImageEditCoordinator` 从 `VertexAIConfig` 表独立加载的配置。
