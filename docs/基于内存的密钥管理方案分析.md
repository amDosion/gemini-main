# 基于内存的密钥管理方案分析

## 📋 方案概述

本方案旨在将敏感密钥（`ENCRYPTION_KEY` 和 `JWT_SECRET_KEY`）存储在**后端进程的内存**中，而不是持久化到文件系统。即使是管理员，也只能通过专用的安全工具来查看密钥，而不能直接访问内存或文件。

**重要说明**：
- ✅ **后端进程内存**：密钥存储在 Python 后端进程的模块级变量中（进程内存）
- ❌ **不是浏览器内存**：与前端/浏览器无关，完全是后端的安全机制
- ❌ **不是文件系统**：密钥不写入任何文件，仅在进程运行时存在于内存中
- ✅ **进程绑定**：密钥与后端进程生命周期绑定，进程重启后需要重新加载

---

## 🎯 方案目标

1. **密钥仅存在于后端进程内存**：密钥不写入文件系统，只在 Python 后端进程的内存中存储（模块级变量）
2. **进程启动时加载**：密钥通过安全通道（环境变量、启动参数、密钥管理服务）在进程启动时加载到内存
3. **管理员工具访问**：提供专用的 CLI 工具，通过安全的 IPC（进程间通信）机制从后端进程获取密钥
4. **最小权限原则**：即使是管理员，也需要通过工具验证身份后才能查看密钥
5. **进程绑定**：密钥与后端进程生命周期绑定，进程终止后密钥从内存中消失

---

## ✅ 可行性分析

### 1. 技术可行性

**✅ 完全可行**

Python 后端进程中可以实现内存存储：

```python
# 模块级别的全局变量（后端进程内存存储）
# 这些变量存储在 Python 解释器的内存空间中，与后端进程绑定
_encryption_key: Optional[str] = None
_jwt_secret_key: Optional[str] = None

def initialize_keys(encryption_key: str, jwt_secret: str):
    """
    进程启动时初始化密钥（仅一次）
    
    密钥存储在 Python 进程的内存中（模块级变量），
    不会写入文件系统，也不会传输到前端/浏览器。
    """
    global _encryption_key, _jwt_secret_key
    _encryption_key = encryption_key
    _jwt_secret_key = jwt_secret
    # 密钥仅存在于后端进程内存中，不会写入文件
    # 进程终止后，密钥从内存中消失

def get_encryption_key() -> str:
    """获取密钥（从后端进程内存）"""
    if _encryption_key is None:
        raise RuntimeError("密钥未初始化")
    return _encryption_key
```

**关键点**：
- 密钥存储在 **Python 后端进程的内存**中（不是浏览器内存）
- 使用模块级变量（`_encryption_key`, `_jwt_secret_key`）
- 密钥与后端进程生命周期绑定
- 进程终止后，密钥从内存中消失

### 2. 安全性分析

**优势**：
- ✅ 密钥不会在文件系统中留下痕迹（后端进程内存存储）
- ✅ 进程崩溃时，密钥不会泄露到核心转储（如果配置正确）
- ✅ 符合"最小权限原则"
- ✅ 减少攻击面（文件系统访问、备份泄露等）
- ✅ 密钥与后端进程绑定，进程终止后自动清除
- ✅ 前端/浏览器无法直接访问后端进程内存中的密钥

**风险**：
- ⚠️ 内存转储（Memory Dump）仍可能获取密钥（需要 root 权限）
- ⚠️ 调试器（Debugger）可以读取后端进程内存（需要进程访问权限）
- ⚠️ 进程重启后需要重新加载密钥
- ⚠️ 多进程部署时，每个后端进程都需要独立加载密钥

### 3. 实现复杂度

**中等复杂度**：
- 需要实现 IPC 机制（Unix socket、命名管道等）
- 需要实现身份验证和授权
- 需要处理进程重启场景
- 需要与现有系统集成

## 🔧 实现方案

### 方案 A：环境变量 + IPC Server（推荐）

**特点**：
- 密钥通过环境变量传入（生产环境使用密钥管理服务注入）
- 进程启动时加载到内存
- IPC Server 提供管理员工具访问接口

**优点**：
- 实现简单
- 兼容现有部署方式
- 支持容器化部署

**缺点**：
- 环境变量可能被其他进程读取（需要进程隔离）

### 方案 B：启动参数 + IPC Server

**特点**：
- 密钥通过命令行参数传入
- 进程启动时加载到内存
- IPC Server 提供管理员工具访问接口

**优点**：
- 更安全（参数不会出现在环境变量列表中）
- 支持密钥管理服务集成

**缺点**：
- 命令行参数可能出现在进程列表中（需要隐藏）

### 方案 C：密钥管理服务集成 + IPC Server（最安全）

**特点**：
- 集成密钥管理服务（如 HashiCorp Vault、AWS Secrets Manager）
- 进程启动时从服务获取密钥
- IPC Server 提供管理员工具访问接口

**优点**：
- 最安全
- 支持密钥轮换
- 支持审计日志

**缺点**：
- 需要额外的密钥管理服务
- 实现复杂度较高

### 方案 D：Client 进程 + Key Service（推荐，最佳安全隔离）

**特点**：
- **Key Service**：独立的密钥管理服务进程，密钥仅存在于该进程内存中
- **Client 进程**：应用进程（FastAPI），通过 IPC 从 Key Service 获取密钥
- **完全隔离**：应用进程无法直接访问密钥，只能通过 IPC 请求
- **独立管理**：Key Service 可以独立重启、更新，不影响应用进程

- 生产环境（高安全要求）

## 📝 详细实现设计

### 方案 D 详细设计：Client 进程 + Key Service

#### 1. Key Service 进程（独立服务）

```python
# backend/services/key_service/main.py

"""
Key Service - 独立的密钥管理服务进程

职责：
1. 管理所有密钥（ENCRYPTION_KEY, JWT_SECRET_KEY）
2. 密钥仅存储在 Key Service 进程内存中
3. 通过 IPC 向 Client 进程提供密钥
4. 提供管理员工具访问接口
"""

import asyncio
import socket
import json
import hashlib
import hmac
import threading
from pathlib import Path
from typing import Optional, Dict
import logging
import os

logger = logging.getLogger(__name__)

# Unix Socket 路径
CLIENT_SOCKET_PATH = Path("/tmp/gemini_key_service_client.sock")
ADMIN_SOCKET_PATH = Path("/tmp/gemini_key_service_admin.sock")

# Key Service 进程内存存储（模块级变量）
_encryption_key: Optional[str] = None
_jwt_secret_key: Optional[str] = None
_keys_initialized = False
_keys_lock = threading.Lock()

def initialize_keys(
    encryption_key: Optional[str] = None,
    jwt_secret_key: Optional[str] = None
) -> None:
    """初始化密钥（Key Service 启动时调用）"""
    global _encryption_key, _jwt_secret_key, _keys_initialized
    
    with _keys_lock:
        if _keys_initialized:
            logger.warning("[KeyService] 密钥已初始化，跳过重复初始化")
            return
        
        # 从环境变量或启动参数加载
        _encryption_key = encryption_key or os.getenv('ENCRYPTION_KEY')
        _jwt_secret_key = jwt_secret_key or os.getenv('JWT_SECRET_KEY')
        
        if not _encryption_key:
            raise RuntimeError("ENCRYPTION_KEY 未设置，无法启动 Key Service")
        
        _keys_initialized = True
        logger.info("[KeyService] ✅ 密钥已初始化（Key Service 进程内存存储）")

def get_encryption_key() -> str:
    """获取 ENCRYPTION_KEY（从 Key Service 进程内存）"""
    if not _keys_initialized or _encryption_key is None:
        raise RuntimeError("ENCRYPTION_KEY 未初始化")
    return _encryption_key

def get_jwt_secret_key() -> str:
    """获取 JWT_SECRET_KEY（从 Key Service 进程内存）"""
    if not _keys_initialized or _jwt_secret_key is None:
        raise RuntimeError("JWT_SECRET_KEY 未初始化")
    return _jwt_secret_key

class KeyServiceServer:
    """Key Service IPC Server"""
    
    def __init__(self, client_token: str, admin_password_hash: str):
        """
        初始化 Key Service
        
        Args:
            client_token: Client 进程认证令牌（用于验证应用进程身份）
            admin_password_hash: 管理员密码哈希
        """
        self.client_token = client_token
        self.admin_password_hash = admin_password_hash
        self.client_socket_path = CLIENT_SOCKET_PATH
        self.admin_socket_path = ADMIN_SOCKET_PATH
        self.running = False
    
    def start(self):
        """启动 Key Service"""
        if self.running:
            return
        
        # 启动 Client IPC Server（应用进程访问）
        client_thread = threading.Thread(
            target=self._run_client_server,
            daemon=True
        )
        client_thread.start()
        
        # 启动 Admin IPC Server（管理员工具访问）
        admin_thread = threading.Thread(
            target=self._run_admin_server,
            daemon=True
        )
        admin_thread.start()
        
        self.running = True
        logger.info("[KeyService] ✅ Key Service 已启动")
        logger.info(f"  - Client Socket: {self.client_socket_path}")
        logger.info(f"  - Admin Socket: {self.admin_socket_path}")
    
    def _run_client_server(self):
        """运行 Client IPC Server（应用进程访问）"""
        # 删除旧的 socket 文件
        if self.client_socket_path.exists():
            self.client_socket_path.unlink()
        
        server_socket = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
        server_socket.bind(str(self.client_socket_path))
        server_socket.listen(5)
        self.client_socket_path.chmod(0o600)
        
        logger.info(f"[KeyService] Client IPC Server 已启动: {self.client_socket_path}")
        
        while self.running:
            try:
                conn, _ = server_socket.accept()
                threading.Thread(
                    target=self._handle_client_request,
                    args=(conn,),
                    daemon=True
                ).start()
            except Exception as e:
                if self.running:
                    logger.error(f"[KeyService] Client 连接失败: {e}")
    
    def _run_admin_server(self):
        """运行 Admin IPC Server（管理员工具访问）"""
        # 删除旧的 socket 文件
        if self.admin_socket_path.exists():
            self.admin_socket_path.unlink()
        
        server_socket = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
        server_socket.bind(str(self.admin_socket_path))
        server_socket.listen(1)
        self.admin_socket_path.chmod(0o600)
        
        logger.info(f"[KeyService] Admin IPC Server 已启动: {self.admin_socket_path}")
        
        while self.running:
            try:
                conn, _ = server_socket.accept()
                self._handle_admin_request(conn)
            except Exception as e:
                if self.running:
                    logger.error(f"[KeyService] Admin 连接失败: {e}")
    
    def _handle_client_request(self, conn: socket.socket):
        """处理 Client 进程请求"""
        try:
            data = conn.recv(4096).decode('utf-8')
            request = json.loads(data)
            
            # 验证 Client 令牌
            if request.get('token') != self.client_token:
                conn.send(json.dumps({'error': '身份验证失败'}).encode())
                return
            
            # 返回请求的密钥
            key_type = request.get('key_type')
            if key_type == 'encryption_key':
                key = get_encryption_key()
            elif key_type == 'jwt_secret_key':
                key = get_jwt_secret_key()
            else:
                conn.send(json.dumps({'error': '无效的密钥类型'}).encode())
                return
            
            response = {'success': True, 'key': key}
            conn.send(json.dumps(response).encode())
            
        except Exception as e:
            logger.error(f"[KeyService] 处理 Client 请求失败: {e}")
            conn.send(json.dumps({'error': str(e)}).encode())
        finally:
            conn.close()
    
    def _handle_admin_request(self, conn: socket.socket):
        """处理管理员工具请求"""
        try:

            if request.get('action') != 'authenticate':
                conn.send(json.dumps({'error': '需要先进行身份验证'}).encode())
                return
            
            # 验证管理员密码
            password = request.get('password', '')
            password_hash = hashlib.sha256(password.encode()).hexdigest()
            
            if not hmac.compare_digest(password_hash, self.admin_password_hash):
                conn.send(json.dumps({'error': '身份验证失败'}).encode())
                return
            
            # 返回所有密钥
            response = {
                'success': True,
                'encryption_key': get_encryption_key(),
                'jwt_secret_key': get_jwt_secret_key()
            }
            conn.send(json.dumps(response).encode())
            
        except Exception as e:
            logger.error(f"[KeyService] 处理 Admin 请求失败: {e}")
            conn.send(json.dumps({'error': str(e)}).encode())
        finally:
            conn.close()
```

```python
# backend/app/core/key_service_client.py

"""
Key Service Client - 应用进程通过此客户端从 Key Service 获取密钥

应用进程不存储密钥，每次需要时从 Key Service 获取。
可以缓存密钥以减少 IPC 调用（但缓存时间应较短）。
"""

import socket
import json
from pathlib import Path
from typing import Optional
import logging
import time
import threading

logger = logging.getLogger(__name__)

# Key Service Socket 路径
KEY_SERVICE_SOCKET = Path("/tmp/gemini_key_service_client.sock")

# 客户端缓存（可选，减少 IPC 调用）
_cache: Dict[str, tuple] = {}  # {key_type: (key_value, expire_time)}
_cache_lock = threading.Lock()
_cache_ttl = 300  # 缓存 5 分钟

class KeyServiceClient:
    """Key Service 客户端"""
    
    def __init__(self, client_token: str):
        """
        初始化客户端
        
        Args:
            client_token: Client 认证令牌（必须与 Key Service 配置一致）
        """
        self.client_token = client_token
        self.socket_path = KEY_SERVICE_SOCKET
    
    def _get_key_from_service(self, key_type: str) -> str:
        """从 Key Service 获取密钥"""
        if not self.socket_path.exists():
            raise RuntimeError(
                f"Key Service 未运行（Socket 不存在: {self.socket_path}）"
            )
        
        try:
            client_socket = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
            client_socket.connect(str(self.socket_path))
            
            request = {
                'token': self.client_token,
                'key_type': key_type
            }
            client_socket.send(json.dumps(request).encode())
            
            response_data = client_socket.recv(4096).decode('utf-8')
            response = json.loads(response_data)
            
            if 'error' in response:
                raise RuntimeError(f"Key Service 错误: {response['error']}")
            
            return response['key']
            
        except Exception as e:
            logger.error(f"[KeyServiceClient] 获取密钥失败: {e}")
            raise
        finally:
            if 'client_socket' in locals():
                client_socket.close()
    
    def get_encryption_key(self, use_cache: bool = True) -> str:
        """获取 ENCRYPTION_KEY"""
        cache_key = 'encryption_key'
        
        # 检查缓存
        if use_cache:
            with _cache_lock:
                if cache_key in _cache:
                    key_value, expire_time = _cache[cache_key]
                    if time.time() < expire_time:
                        logger.debug("[KeyServiceClient] 使用缓存的 ENCRYPTION_KEY")
                        return key_value
                    else:
                        # 缓存过期，删除
                        del _cache[cache_key]
        
        # 从 Key Service 获取
        key = self._get_key_from_service('encryption_key')
        
        # 更新缓存
        if use_cache:
            with _cache_lock:
                _cache[cache_key] = (key, time.time() + _cache_ttl)
        
        return key
    
    def get_jwt_secret_key(self, use_cache: bool = True) -> str:
        """获取 JWT_SECRET_KEY"""
        cache_key = 'jwt_secret_key'
        
        # 检查缓存
        if use_cache:
            with _cache_lock:
                if cache_key in _cache:
                    key_value, expire_time = _cache[cache_key]
                    if time.time() < expire_time:
                        logger.debug("[KeyServiceClient] 使用缓存的 JWT_SECRET_KEY")
                        return key_value
                    else:
                        # 缓存过期，删除
                        del _cache[cache_key]
        
        # 从 Key Service 获取
        key = self._get_key_from_service('jwt_secret_key')
        
        # 更新缓存
        if use_cache:
            with _cache_lock:
                _cache[cache_key] = (key, time.time() + _cache_ttl)
        
        return key

# 全局客户端实例（应用进程启动时初始化）
_key_service_client: Optional[KeyServiceClient] = None

def initialize_key_service_client(client_token: str):
    """初始化 Key Service 客户端（应用进程启动时调用）"""
    global _key_service_client
    _key_service_client = KeyServiceClient(client_token)
    logger.info("[KeyServiceClient] ✅ Key Service 客户端已初始化")

def get_encryption_key() -> str:
    """获取 ENCRYPTION_KEY（从 Key Service）"""
    if _key_service_client is None:
        raise RuntimeError("Key Service 客户端未初始化")
    return _key_service_client.get_encryption_key()

def get_jwt_secret_key() -> str:
    """获取 JWT_SECRET_KEY（从 Key Service）"""
    if _key_service_client is None:
        raise RuntimeError("Key Service 客户端未初始化")
    return _key_service_client.get_jwt_secret_key()
```

```python
# backend/app/core/key_viewer_server.py

import socket
import json
import hashlib
import hmac
import threading
from pathlib import Path
from typing import Optional
import logging

logger = logging.getLogger(__name__)

# Unix Socket 路径
SOCKET_PATH = Path("/tmp/gemini_key_viewer.sock")

class KeyViewerServer:
    """密钥查看服务（IPC Server）"""
    
    def __init__(self, admin_password_hash: str):
        """
        初始化 IPC Server
        
        Args:
            admin_password_hash: 管理员密码的 SHA256 哈希值
        """
        self.admin_password_hash = admin_password_hash
        self.socket_path = SOCKET_PATH
        self.server_socket: Optional[socket.socket] = None
        self.running = False
        self.thread: Optional[threading.Thread] = None
    
    def start(self):
        """启动 IPC Server"""
        if self.running:
            return
        
        # 删除旧的 socket 文件（如果存在）
        if self.socket_path.exists():
            self.socket_path.unlink()
        
        # 创建 Unix Socket
        self.server_socket = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
        self.server_socket.bind(str(self.socket_path))
        self.server_socket.listen(1)
        
        # 设置文件权限（仅所有者可访问）
        self.socket_path.chmod(0o600)
        
        self.running = True
        self.thread = threading.Thread(target=self._run, daemon=True)
        self.thread.start()
        
        logger.info(f"[KeyViewerServer] ✅ IPC Server 已启动: {self.socket_path}")
    
    def stop(self):
        """停止 IPC Server"""
        self.running = False
        if self.server_socket:
            self.server_socket.close()
        if self.socket_path.exists():
            self.socket_path.unlink()
        logger.info("[KeyViewerServer] IPC Server 已停止")
    
    def _run(self):
        """运行 IPC Server（后台线程）"""
        while self.running:
            try:
                conn, addr = self.server_socket.accept()
                self._handle_client(conn)
            except Exception as e:
                if self.running:
                    logger.error(f"[KeyViewerServer] 处理客户端连接失败: {e}")
    
    def _handle_client(self, conn: socket.socket):
        """处理客户端请求"""
        try:
            # 1. 接收身份验证请求
            data = conn.recv(4096).decode('utf-8')
            request = json.loads(data)
            
            if request.get('action') != 'authenticate':
                conn.send(json.dumps({'error': '需要先进行身份验证'}).encode())
                return
            
            # 2. 验证密码
            password = request.get('password', '')
            password_hash = hashlib.sha256(password.encode()).hexdigest()
            
            if not hmac.compare_digest(password_hash, self.admin_password_hash):
                conn.send(json.dumps({'error': '身份验证失败'}).encode())
                return
            
            # 3. 返回密钥（加密传输）
            from .memory_key_manager import get_encryption_key, get_jwt_secret_key
            
            response = {
                'success': True,
                'encryption_key': get_encryption_key(),
                'jwt_secret_key': get_jwt_secret_key()
            }
            
            conn.send(json.dumps(response).encode())
            
        except Exception as e:
            logger.error(f"[KeyViewerServer] 处理客户端请求失败: {e}")
            conn.send(json.dumps({'error': str(e)}).encode())
        finally:
            conn.close()
```

## 🔐 安全性增强措施

### 1. 身份验证

**方案 A：密码验证**
- 管理员密码存储在环境变量 `ADMIN_VIEW_KEY_PASSWORD`
- 使用 SHA256 哈希比较（防止时序攻击）

**方案 B：证书验证（更安全）**
- 使用客户端证书（TLS）
- 仅允许特定证书访问

**方案 C：OTP（一次性密码）**
- 使用 TOTP（Time-based OTP）
- 每次查看需要新的 OTP

### 2. 传输加密

- 使用 TLS 加密 Unix Socket 通信
- 或使用 Fernet 加密响应数据

### 3. 访问控制

- 限制 Unix Socket 文件权限（`0o600`）
- 记录所有访问日志
- 支持 IP 白名单（如果使用 TCP Socket）

### 4. 后端进程内存保护

- 使用 `mlock()` 锁定后端进程内存（防止交换到磁盘）
- 使用 `mprotect()` 设置内存保护标志
- 后端进程崩溃时自动清理内存
- 确保密钥不会出现在核心转储文件中

## ⚖️ 优缺点对比

### 优点

1. **更高的安全性**
   - 密钥不存储在文件系统中（仅在后端进程内存中）
   - 减少文件泄露风险
   - 符合安全最佳实践
   - 密钥与后端进程绑定，进程终止后自动清除
   - 前端/浏览器无法直接访问后端进程内存

2. **最小权限原则**
   - 即使是管理员，也需要通过工具验证身份
   - 支持访问审计

3. **灵活性**
   - 支持多种密钥来源（环境变量、密钥管理服务等）
   - 易于集成到现有系统

### 缺点

1. **实现复杂度**
   - 需要实现 IPC 机制
   - 需要处理进程重启场景
   - 需要身份验证和授权

2. **运维复杂度**
   - 进程重启后需要重新加载密钥
   - 多进程部署时，每个进程都需要独立加载
   - 需要管理管理员密码/证书

3. **可用性**
   - 如果密钥管理服务不可用，进程无法启动
   - 需要额外的监控和告警

4. **后端进程内存风险**
   - 内存转储仍可能获取密钥（需要 root 权限访问后端进程）
   - 调试器可以读取后端进程内存（需要进程访问权限）
   - 需要额外的内存保护措施（mlock、mprotect）
   - 多进程部署时，每个后端进程都有独立的密钥副本

## ⚠️ 注意事项

1. **后端进程重启**
   - 后端进程重启后，密钥需要重新加载到进程内存
   - 需要确保密钥来源（环境变量、密钥管理服务）可用
   - 进程终止时，密钥从内存中自动清除

2. **多进程部署**
   - 每个后端进程都需要独立加载密钥到自己的进程内存
   - 每个进程有独立的密钥副本（存储在各自的 Python 解释器内存中）
   - 考虑使用共享内存（如果支持，但会增加复杂度）

3. **容器化部署**
   - 环境变量可以通过 Kubernetes Secrets 注入
   - 或使用密钥管理服务的 Sidecar 模式

4. **备份和恢复**
   - 密钥不存储在文件中，无法直接备份
   - 需要从密钥管理服务恢复
   - 或使用安全的密钥备份机制

5. **调试和开发**
   - 开发环境可能需要文件存储（方便调试）
   - 可以保留文件存储作为开发环境的回退方案
   - 生产环境使用后端进程内存存储（更安全）

6. **与前端的关系**
   - 密钥存储在后端进程内存中，与前端/浏览器完全隔离
   - 前端无法直接访问后端进程内存
   - 前端只能通过 API 调用后端，后端使用内存中的密钥处理请求
