# 代码审查

对指定的代码或文件进行全面的代码审查。

## 审查范围

### 安全性检查
1. **认证和授权**
   - 敏感端点是否使用 `require_current_user`
   - 数据查询是否加 `user_id` 过滤
   - API Key 是否安全存储和传输

2. **输入验证**
   - 是否使用 Pydantic 模型验证
   - 是否存在 SQL 注入风险
   - 是否存在 XSS 风险

3. **敏感数据**
   - API Key 是否暴露在日志中
   - 响应是否返回敏感字段

### 代码质量检查
1. **类型安全**
   - 是否有完整的类型注解
   - 是否使用 `any` 类型

2. **错误处理**
   - 是否有适当的 try-catch
   - 错误是否正确传播和处理

3. **代码风格**
   - 命名是否符合约定
   - 是否遵循项目结构

### 性能检查
1. **React 组件**
   - 是否使用 `useCallback`/`useMemo`
   - 是否有不必要的重渲染
   - 是否有内存泄漏风险

2. **数据库查询**
   - 是否有 N+1 查询问题
   - 是否有必要的索引
   - 是否有事务处理

## 输出格式

```markdown
## 代码审查报告

### 文件：[文件路径]

### 严重问题 (必须修复)
- [ ] **[问题标题]**
  - 位置：第 X 行
  - 问题：[问题描述]
  - 风险：[安全/性能/功能]
  - 建议：[修复方案]

### 警告 (建议修复)
- [ ] **[问题标题]**
  - 位置：第 X 行
  - 问题：[问题描述]
  - 建议：[改进方案]

### 建议 (可选优化)
- [ ] **[优化建议]**
  - 位置：第 X 行
  - 理由：[为什么这样做更好]

### 优点
- [代码中做得好的地方]

### 总结
[总体评价和关键改进点]
```
