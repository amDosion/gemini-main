# 路由与逻辑完全分离架构方案

## 问题分析

### 当前架构问题

1. **路由包含业务逻辑**：

                                                                                                                                                                                                - `generate.py` 包含 image-gen, image-edit, video-gen, audio-gen 等多种模式的逻辑
                                                                                                                                                                                                - `image_edit.py` 包含图片编辑的业务逻辑
                                                                                                                                                                                                - `image_expand.py` 包含图片扩展的业务逻辑
                                                                                                                                                                                                - 路由文件既负责路由又负责业务处理

2. **路由分散**：

                                                                                                                                                                                                - `/api/generate/{provider}/image` - 图片生成
                                                                                                                                                                                                - `/api/generate/{provider}/image/edit` - 图片编辑
                                                                                                                                                                                                - `/api/generate/{provider}/video` - 视频生成
                                                                                                                                                                                                - `/api/generate/{provider}/speech` - 音频生成
                                                                                                                                                                                                - `/api/generate/tongyi/image` - Tongyi 专用
                                                                                                                                                                                                - `/api/tryon` - 虚拟试穿
                                                                                                                                                                                                - `/api/google/outpaint` - Google 扩展
                                                                                                                                                                                                - `/api/qwen/search` - Qwen 搜索
                                                                                                                                                                                                - 路由不统一，难以维护

3. **前端模式与后端路由不匹配**：

                                                                                                                                                                                                - 前端有 15 种模式（AppMode）
                                                                                                                                                                                                - 后端路由分散在多个文件中
                                                                                                                                                                                                - 没有统一的模式到路由映射

### 实施步骤

#### 阶段 1: 创建服务分发层

**文件**: `backend/app/services/mode_dispatcher.py` (新建)

创建统一的模式分发器，根据 provider + mode 分发到对应服务：

```python
"""
模式服务分发器

根据 provider 和 mode 参数，分发到对应的服务实例。
路由层只负责调用分发器，不包含任何业务逻辑。
"""
from typing import Optional, Dict, Any
from sqlalchemy.orm import Session
from .provider_factory import ProviderFactory
import logging

logger = logging.getLogger(__name__)


class ModeDispatcher:
    """模式服务分发器"""
    
    @staticmethod
    async def dispatch(
        provider: str,
        mode: str,
        api_key: str,
        api_url: Optional[str] = None,
        user_id: Optional[str] = None,
        db: Optional[Session] = None,
        **kwargs
    ):
        """
        根据 provider 和 mode 分发到对应服务
        
        Args:
            provider: 提供商名称 (google, tongyi, openai, etc.)
            mode: 模式名称 (image-gen, image-edit, video-gen, etc.)
            api_key: API 密钥
            api_url: 自定义 API URL
            user_id: 用户 ID
            db: 数据库会话
            **kwargs: 其他参数
            
        Returns:
            服务实例
            
        Raises:
            ValueError: 如果 provider 或 mode 不支持
        """
        # 模式到服务的映射
        mode_service_map = {
            # 图片相关
            "image-gen": "ImageGenService",
            "image-chat-edit": "ImageChatEditService",
            "image-mask-edit": "ImageMaskEditService",
            "image-inpainting": "ImageInpaintingService",
            "image-background-edit": "ImageBackgroundEditService",
            "image-recontext": "ImageRecontextService",
            "image-edit": "ImageEditService",
            "image-outpainting": "ImageOutpaintingService",
            
            # 视频和音频
            "video-gen": "VideoGenService",
            "audio-gen": "AudioGenService",
            
            # 其他
            "pdf-extract": "PdfExtractService",
            "virtual-try-on": "VirtualTryOnService",
            "deep-research": "DeepResearchService",
            "multi-agent": "MultiAgentService",
        }
        
        service_class_name = mode_service_map.get(mode)
        if not service_class_name:
            raise ValueError(f"Unsupported mode: {mode}")
        
        # 根据 provider 和 mode 创建服务
        return await ModeDispatcher._create_service(
            provider=provider,
            mode=mode,
            service_class_name=service_class_name,
            api_key=api_key,
            api_url=api_url,
            user_id=user_id,
            db=db,
            **kwargs
        )
    
    @staticmethod
    async def _create_service(
        provider: str,
        mode: str,
        service_class_name: str,
        api_key: str,
        api_url: Optional[str] = None,
        user_id: Optional[str] = None,
        db: Optional[Session] = None,
        **kwargs
    ):
        """创建具体的服务实例"""
        
        # 特殊处理：某些模式需要专用服务
        if mode == "image-chat-edit" and provider == "google":
            from .gemini.conversational_image_edit_service import ConversationalImageEditService
            return ConversationalImageEditService(api_key=api_key, user_id=user_id, db=db)
        
        if mode == "image-gen" and provider == "tongyi":
            from .tongyi.image_service import TongyiImageService
            return TongyiImageService.get_instance(api_key)
        
        # 默认：使用 ProviderFactory
        return ProviderFactory.create(
            provider=provider,
            api_key=api_key,
            api_url=api_url,
            user_id=user_id,
            db=db,
            **kwargs
        )
```

#### 阶段 2: 创建统一模式路由

**文件**: `backend/app/routers/modes.py` (新建)

创建统一的模式路由，只负责请求接收和参数验证：

```python
"""
统一模式路由

所有非聊天模式都通过此路由处理。
路由层只负责：
1. 接收请求
2. 参数验证
3. 用户认证
4. 调用服务分发器
5. 返回响应

不包含任何业务逻辑。
"""
from fastapi import APIRouter, HTTPException, Request, Depends
from pydantic import BaseModel
from typing import Optional, Dict, Any, List
from sqlalchemy.orm import Session
from ..core.database import get_db
from ..core.dependencies import require_current_user
from ..core.credential_manager import get_provider_credentials
from ..services.mode_dispatcher import ModeDispatcher
import logging

logger = logging.getLogger(__name__)

router = APIRouter(prefix="/api/modes", tags=["modes"])


# ==================== 统一请求模型 ====================

class ModeRequest(BaseModel):
    """统一的模式请求模型"""
    mode: str  # 模式名称
    modelId: Optional[str] = None
    prompt: Optional[str] = None
    options: Optional[Dict[str, Any]] = None
    apiKey: Optional[str] = None
    # 其他通用字段...
    attachments: Optional[List[Dict[str, Any]]] = None


# ==================== 统一路由端点 ====================

@router.post("/{provider}/{mode}")
async def handle_mode(
    provider: str,
    mode: str,
    request_body: ModeRequest,
    request: Request,
    user_id: str = Depends(require_current_user),  # ✅ 统一认证
    db: Session = Depends(get_db)
):
    """
    统一的模式处理端点
    
    根据 provider 和 mode 参数，自动分发到对应的服务。
    
    Args:
        provider: 提供商名称 (google, tongyi, openai, etc.)
        mode: 模式名称 (image-gen, image-edit, video-gen, etc.)
        request_body: 请求体
        request: FastAPI 请求对象
        user_id: 用户 ID（通过依赖注入自动获取）
        db: 数据库会话
        
    Returns:
        服务处理结果
    """
    try:
        # ✅ 1. 参数验证
        if not provider or not mode:
            raise HTTPException(status_code=400, detail="Provider and mode are required")
        
        # ✅ 2. 获取凭证（统一处理）
        api_key, api_url = await get_provider_credentials(
            provider=provider,
            db=db,
            user_id=user_id,
            request_api_key=request_body.apiKey,
            request_base_url=request_body.options.get("baseUrl") if request_body.options else None
        )
        
        # ✅ 3. 调用服务分发器（路由层不包含业务逻辑）
        service = await ModeDispatcher.dispatch(
            provider=provider,
            mode=mode,
            api_key=api_key,
            api_url=api_url,
            user_id=user_id,
            db=db
        )
        
        # ✅ 4. 调用服务的处理方法（业务逻辑在服务层）
        result = await service.execute(
            mode=mode,
            request_data=request_body.dict(),
            **request_body.options or {}
        )
        
        # ✅ 5. 返回响应（统一格式）
        return {
            "success": True,
            "data": result,
            "provider": provider,
            "mode": mode
        }
    
    except ValueError as e:
        logger.warning(f"[Modes] Invalid request: {e}")
        raise HTTPException(status_code=400, detail=str(e))
    except HTTPException:
        raise
    except Exception as e:
        logger.error(f"[Modes] Error processing {provider}/{mode}: {e}", exc_info=True)
        raise HTTPException(status_code=500, detail=f"Internal server error: {str(e)}")
```

#### 阶段 4: 创建服务基类

**文件**: `backend/app/services/base_mode_service.py` (新建)

定义所有模式服务的基类：

```python
"""
模式服务基类

所有模式服务都应该继承此基类，实现统一的接口。
"""
from abc import ABC, abstractmethod
from typing import Dict, Any, Optional
from sqlalchemy.orm import Session


class BaseModeService(ABC):
    """模式服务基类"""
    
    def __init__(
        self,
        api_key: str,
        api_url: Optional[str] = None,
        user_id: Optional[str] = None,
        db: Optional[Session] = None
    ):
        self.api_key = api_key
        self.api_url = api_url
        self.user_id = user_id
        self.db = db
    
    @abstractmethod
    async def execute(
        self,
        mode: str,
        request_data: Dict[str, Any],
        **kwargs
    ) -> Dict[str, Any]:
        """
        执行模式操作
        
        Args:
            mode: 模式名称
            request_data: 请求数据
            **kwargs: 其他参数
            
        Returns:
            处理结果
        """
        pass
```

### 架构优势

1. **清晰分层**：

                                                                                                                                                                                                - 路由层：只负责路由
                                                                                                                                                                                                - 分发层：只负责分发
                                                                                                                                                                                                - 服务层：只负责业务逻辑

2. **易于扩展**：

                                                                                                                                                                                                - 新增模式：只需添加服务实现
                                                                                                                                                                                                - 新增提供商：只需在分发器中添加映射

3. **统一认证**：

                                                                                                                                                                                                - 所有路由使用相同的认证依赖
                                                                                                                                                                                                - 认证逻辑集中管理

4. **统一错误处理**：

                                                                                                                                                                                                - 所有路由使用相同的错误处理
                                                                                                                                                                                                - 错误响应格式统一

5. **易于测试**：

                                                                                                                                                                                                - 路由层可以单独测试
                                                                                                                                                                                                - 服务层可以单独测试
                                                                                                                                                                                                - 分发层可以单独测试
