# 修

复模型选择导致的Canvas闪烁问题（正确方案）

## 问题分析

### 问题现象

点击 Header 中的模型选择器后，`ImageEditView.tsx` 和 `ImageExpandView.tsx` 中的 "RIGHT MAIN: Result / Canvas" 区域会出现闪烁。

### 根本原因

1. **模型选择流程**：

- 用户点击模型选择器 → `onModelSelect(model.id)` 
- → `handleModelSelect(id)` → `setCurrentModelId(id)`
- → `useModels` hook 中的 `activeModelConfig` 通过 `useMemo` 更新
- → `activeModelConfig` 作为 prop 传递到视图组件

2. **问题所在**：

- `activeModelConfig` 在 `ImageEditView` 中仅用于：
    - `sidebarContent` 中显示模型名称（第 180 行：`activeModelConfig?.name`）
    - `bottomContent` 中传递给 `InputArea`（第 357 行）
- `activeModelConfig` 在 `ImageExpandView` 中仅用于：
    - `bottomContent` 中传递给 `InputArea`（第 333 行）
- **`mainContent`（Canvas 区域）完全不依赖 `activeModelConfig`**
- 但当 `activeModelConfig` 作为 prop 变化时，整个组件会重新渲染
- 组件重新渲染 → `useImageCanvas` hook 重新调用 → 即使函数引用稳定，对象引用可能变化 → `mainContent` 的 `useMemo` 可能重新计算 → DOM 重新渲染 → 闪烁

3. **关键问题**：

- 切换模型不应该影响 Canvas 区域的渲染
- 但 React 的默认行为是：任何 prop 变化都会导致组件重新渲染
- 需要使用 `React.memo` 来优化，只在真正需要的 prop 变化时重新渲染

### 代码位置

- [frontend/components/views/ImageEditView.tsx](frontend/components/views/ImageEditView.tsx) - 组件定义和 mainContent useMemo
- [frontend/components/views/ImageExpandView.tsx](frontend/components/views/ImageExpandView.tsx) - 组件定义和 mainContent useMemo
- [frontend/components/layout/Header.tsx](frontend/components/layout/Header.tsx) - 第 266-269 行（onModelSelect 调用）

## 解决方案

### 方案：使用 React.memo 优化组件

使用 `React.memo` 包装 `ImageEditView` 和 `ImageExpandView`，自定义比较函数，让组件只在真正影响渲染的 prop 变化时重新渲染。**对于 ImageEditView**：

- `mainContent` 依赖：`loadingState`, `isCompareMode`, `originalImageUrl`, `activeImageUrl`, `canvas.*`, `onImageClick`, `onExpandImage`, `activeAttachments`
- `sidebarContent` 依赖：`messages`, `loadingState`, `activeModelConfig?.name`, `activeImageUrl`, `activeAttachments`
- `bottomContent` 依赖：`handleSend`, `loadingState`, `onStop`, `activeModelConfig`, `setAppMode`, `initialPrompt`, `activeAttachments`, `providerId`

**对于 ImageExpandView**：

- `mainContent` 依赖：`loadingState`, `isCompareMode`, `originalImageUrl`, `activeImageUrl`, `canvas.*`, `onImageClick`, `activeAttachments`
- `sidebarContent` 依赖：`messages`, `loadingState`, `activeImageUrl`, `activeAttachments`
- `bottomContent` 依赖：`handleSend`, `loadingState`, `onStop`, `activeModelConfig`, `setAppMode`, `activeAttachments`, `providerId`, `activeImageUrl`

**优化策略**：

1. 使用 `React.memo` 包装组件
2. 自定义比较函数，对于 `activeModelConfig`，只比较 `id` 或 `name`，而不是整个对象
3. 确保其他关键 prop 的比较逻辑正确
