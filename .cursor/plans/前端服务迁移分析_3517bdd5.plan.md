# 前端服务迁移分析

## 分析目标

根据统一 Provider 架构原则（后端作为 SSOT），识别 `frontend/services` 目录中可以迁移到后端的代码。

## 核心原则

1. **后端是 SSOT**：Provider 配置、客户端创建、路由选择都在后端
2. **前端是薄客户端**：只负责 UI 编排和调用后端 API
3. **前端不应直接实例化 Provider SDK**：所有 SDK 调用应通过后端 API

## 可迁移代码分类

### 一、Google Provider SDK 直接调用（高优先级迁移）

#### 1.1 SDK 实例化代码

- **文件**: `frontend/services/providers/google/utils.ts`
- `createGoogleClient()` - 创建 GoogleGenAI 客户端实例
- `getSdkOptions()` - SDK 配置解析
- **迁移建议**: 这些函数应完全移除，所有 Google SDK 调用应通过后端 API

#### 1.2 图片生成相关

- **文件**: `frontend/services/providers/google/media/image-gen.ts`
- `generateTextToImage()` - 直接调用 `ai.models.generateImages()`
- 文件已标记为 `@deprecated`，但仍在被使用
- **迁移建议**: 完全移除，使用 `UnifiedProviderClient` 或后端 API

- **文件**: `frontend/services/providers/google/media/image-edit.ts`
- `editImage()` - 直接使用 GoogleGenAI SDK 进行图片编辑
- 包含 Google Files API 上传逻辑
- **迁移建议**: 迁移到后端，通过 `/api/image/edit` 或类似端点

#### 1.3 其他媒体功能

- **文件**: `frontend/services/providers/google/media/video.ts`
- `generateVideo()` - 直接调用 Google SDK
- **迁移建议**: 迁移到后端 API

- **文件**: `frontend/services/providers/google/media/audio.ts`
- `generateSpeech()` - 直接调用 Google SDK
- **迁移建议**: 迁移到后端 API

- **文件**: `frontend/services/providers/google/media/virtual-tryon.ts`
- Virtual Try-On 相关功能，直接使用 Google SDK
- **迁移建议**: 迁移到后端（已有部分后端支持，见 `backend/app/routers/tryon.py`）

#### 1.4 Google Files API

- **文件**: `frontend/services/providers/google/fileService.ts`
- `GoogleFileService.uploadFile()` - 直接调用 Google Files API
- `GoogleFileService.getFile()` - 获取文件信息
- **迁移建议**: 迁移到后端 API，前端只传递文件对象

#### 1.5 Google Media Strategy

- **文件**: `frontend/services/providers/google/media/index.ts`
- `googleMediaStrategy` - 所有 Google 媒体功能的入口
- 所有方法都直接创建 GoogleGenAI 客户端
- **迁移建议**: 完全移除，所有调用应通过 `UnifiedProviderClient` 或后端 API

### 二、Tongyi/DashScope Provider SDK 调用（中优先级迁移）

#### 2.1 图片生成

- **文件**: `frontend/services/providers/tongyi/image-gen.ts`
- `generateDashScopeImage()` - 直接调用 DashScope API
- `generateZImage()`, `generateWanV2Image()`, `generateQwenImage()` - 各种模型的生成函数
- **迁移建议**: 迁移到后端（部分已迁移，见 `backend/app/routers/image_edit.py`）

#### 2.2 图片编辑

- **文件**: `frontend/services/providers/tongyi/image-edit.ts`
- `editWanxImage()` - 直接调用 DashScope API
- **迁移建议**: 迁移到后端（已有后端实现）

#### 2.3 图片扩展

- **文件**: `frontend/services/providers/tongyi/image-expand.ts`
- `outPaintWanxImage()` - 直接调用 DashScope API
- **迁移建议**: 迁移到后端

#### 2.4 DashScope API 工具函数

- **文件**: `frontend/services/providers/tongyi/api.ts`
- `uploadDashScopeFile()` - 文件上传（已通过后端代理，但逻辑在前端）
- `submitSync()`, `submitAndPoll()` - API 调用封装
- `resolveDashUrl()` - URL 解析逻辑
- **迁移建议**: 这些工具函数应迁移到后端，前端只调用后端 API

#### 2.5 DashScope Provider

- **文件**: `frontend/services/providers/tongyi/DashScopeProvider.ts`
- `generateImage()` - 调用前端 image-gen.ts
- `outPaintImage()` - 调用前端 image-expand.ts
- `uploadFile()` - 调用前端 api.ts
- **注意**: `sendMessageStream()` 已通过后端（`/api/chat/tongyi`）
- **迁移建议**: 图片相关功能迁移到后端，Provider 类只保留接口定义

### 三、OpenAI Provider（低优先级，需评估）

#### 3.1 OpenAI Provider

- **文件**: `frontend/services/providers/openai/OpenAIProvider.ts`
- `getAvailableModels()` - 直接调用 OpenAI API
- `sendMessageStream()` - 直接调用 OpenAI API
- `generateImage()` - 直接调用 OpenAI API
- **评估**: 
- OpenAI Provider 是通用的 OpenAI-compatible 客户端
- 可能用于自定义代理或第三方服务
- **建议**: 保留但标记为"通用客户端"，明确说明不适用于 SSOT 管理的 Provider

### 四、工厂和路由逻辑（需要简化）

#### 4.1 LLMFactory

- **文件**: `frontend/services/LLMFactory.ts`
- 当前逻辑：根据 providerId 选择 Provider 实例
- **问题**: 部分 Provider（如 Google）已使用 `UnifiedProviderClient`，但 Tongyi 仍使用 `DashScopeProvider`
- **迁移建议**: 
- 统一使用 `UnifiedProviderClient` 作为默认路径
- `DashScopeProvider` 仅保留 chat 功能（已通过后端）
- 图片功能迁移到后端后，`DashScopeProvider` 可完全移除

#### 4.2 Provider 接口

- **文件**: `frontend/services/providers/interfaces.ts`
- `ILLMProvider` 接口定义
- **评估**: 接口定义应保留，但实现应全部通过后端 API

## 迁移优先级

### P0 - 立即迁移（违反架构原则）

1. ✅ `providers/google/utils.ts` - SDK 实例化
2. ✅ `providers/google/media/*` - 所有 Google 媒体功能
3. ✅ `providers/google/fileService.ts` - Google Files API

### P1 - 高优先级（已有部分后端支持）

1. ✅ `providers/tongyi/image-gen.ts` - 图片生成
2. ✅ `providers/tongyi/image-edit.ts` - 图片编辑（后端已有）
3. ✅ `providers/tongyi/image-expand.ts` - 图片扩展
4. ✅ `providers/tongyi/api.ts` - API 工具函数

### P2 - 中优先级（需要统一）

1. ⚠️ `LLMFactory.ts` - 简化 Provider 选择逻辑
2. ⚠️ `DashScopeProvider.ts` - 移除图片相关功能，只保留接口

### P3 - 低优先级（保留但标记）

1. ⚠️ `OpenAIProvider.ts` - 保留作为通用客户端，明确使用场景

## 迁移后的架构

### 前端职责（保留）

- UI 编排和用户交互
- 调用后端 API（`/api/chat/*`, `/api/image/*`, `/api/models/*`）
- 流式响应处理（SSE）
- 文件上传到后端（不直接上传到 Provider）

### 后端职责（新增/增强）

- Provider SDK 实例化和管理
- 所有 Provider API 调用
- 文件上传到 Provider（Google Files API, DashScope OSS）
- 统一错误处理和重试逻辑
- 能力检测和路由选择

## 注意事项

1. **向后兼容**: 迁移过程中需要保持向后兼容，逐步迁移
2. **错误处理**: 后端 API 应提供清晰的错误信息
3. **性能**: 文件上传可能需要通过后端代理（CORS 限制）
4. **测试**: 每个迁移步骤都需要充分测试
