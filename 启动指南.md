# 项目启动指南

## 快速启动

### 方式一：完整功能（推荐）

如果你需要使用**扩图、搜索、PDF 提取**等高级功能，需要同时启动前端和后端。

#### 1. 启动后端服务器

打开第一个终端窗口：

```bash
# 进入后端目录
cd backend

# 启动后端服务（端口 8000）
python -m uvicorn app.main:app --reload --port 8000
```

**预期输出**：
```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

#### 2. 启动前端开发服务器

打开第二个终端窗口：

```bash
# 在项目根目录
npm run dev
```

**预期输出**：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

#### 3. 访问应用

在浏览器中打开：`http://localhost:5173`

---

### 方式二：仅前端（基础功能）

如果你只需要**基础聊天功能**，可以只启动前端。

```bash
# 在项目根目录
npm run dev
```

**功能限制**：
- ✅ 基础聊天功能
- ✅ 配置管理（使用 LocalStorage）
- ❌ 图片扩展功能（需要后端）
- ❌ 网页搜索功能（需要后端）
- ❌ PDF 提取功能（需要后端）

## 详细说明

### 后端服务器

**作用**：
1. **代理 DashScope API**：绕过浏览器 CORS 限制
2. **数据库服务**：提供 PostgreSQL/SQLite 数据存储
3. **高级功能**：搜索、PDF 提取、RAG 等

**端口**：`8000`

**健康检查**：
```bash
# 检查后端是否正常运行
curl http://localhost:8000/health
```

**预期响应**：
```json
{
  "status": "healthy",
  "services": {
    "selenium": false,
    "pdf_extraction": false,
    "embedding": false
  }
}
```

---

### 前端开发服务器

**作用**：
1. **提供 Web UI**：React 应用界面
2. **热重载**：代码修改后自动刷新
3. **代理配置**：自动转发 `/api` 请求到后端

**端口**：`5173`（Vite 默认）

**代理配置**（`vite.config.ts`）：
```typescript
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true,
    secure: false,
  }
}
```

这意味着：
- 前端请求 `/api/dashscope/...` 
- 自动转发到 `http://localhost:8000/api/dashscope/...`

## 常见问题

### Q1: 后端启动失败

**错误**：`ModuleNotFoundError: No module named 'fastapi'`

**解决方案**：安装 Python 依赖
```bash
cd backend
pip install -r requirements.txt
```

---

### Q2: 端口被占用

**错误**：`Address already in use`

**解决方案**：

**Windows**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :8000

# 结束进程（替换 PID）
taskkill /PID <进程ID> /F
```

**或者使用其他端口**：
```bash
# 后端使用 8001 端口
python -m uvicorn app.main:app --reload --port 8001
```

然后修改 `vite.config.ts` 中的代理配置：
```typescript
proxy: {
  '/api': {
    target: 'http://localhost:8001',  // 改为 8001
    changeOrigin: true,
    secure: false,
  }
}
```

---

### Q3: 前端无法连接后端

**症状**：浏览器控制台显示 `Failed to fetch`

**检查清单**：

1. **后端是否运行**？
   ```bash
   curl http://localhost:8000/health
   ```

2. **端口是否正确**？
   - 后端默认：`8000`
   - 前端默认：`5173`

3. **防火墙是否阻止**？
   - 临时关闭防火墙测试
   - 或添加端口例外

4. **Vite 代理配置是否正确**？
   - 检查 `vite.config.ts` 中的 `target`

---

### Q4: 扩图功能不可用

**症状**：点击扩图后显示错误

**原因**：后端未启动

**解决方案**：
1. 启动后端服务器（见上文）
2. 确认后端正常运行：`curl http://localhost:8000/health`
3. 刷新浏览器页面

**技术说明**：
- 浏览器环境中，DashScope API 调用**必须**通过后端代理
- 这是因为浏览器的 CORS 安全策略
- 没有后端，扩图功能无法工作

## 生产环境部署

### 构建前端

```bash
# 构建生产版本
npm run build

# 输出目录：dist/
```

### 部署后端

```bash
# 使用 Gunicorn（推荐）
cd backend
gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

### 使用 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 开发工作流

### 日常开发

1. **启动后端**（一次性，保持运行）
   ```bash
   cd backend
   python -m uvicorn app.main:app --reload --port 8000
   ```

2. **启动前端**（一次性，保持运行）
   ```bash
   npm run dev
   ```

3. **开始开发**
   - 修改代码
   - 浏览器自动刷新
   - 查看效果

### 调试技巧

**前端调试**：
- 打开浏览器开发者工具（F12）
- 查看 Console 标签页的日志
- 查看 Network 标签页的网络请求

**后端调试**：
- 查看终端输出的日志
- 使用 `print()` 或 `logger.info()` 添加日志
- 使用 FastAPI 自动生成的文档：`http://localhost:8000/docs`

## 环境变量配置

### 前端环境变量

创建 `.env.local` 文件（项目根目录）：

```bash
# DashScope API Key（可选，也可以在 UI 中配置）
VITE_DASHSCOPE_API_KEY=sk-your-api-key-here
```

### 后端环境变量

创建 `backend/.env` 文件：

```bash
# 数据库配置
DATABASE_URL=postgresql://user:password@localhost:5432/dbname

# 或使用 SQLite（默认）
DATABASE_URL=sqlite:///./test.db

# DashScope API Key（可选）
DASHSCOPE_API_KEY=sk-your-api-key-here
```
