1. 前端 UI AI 输出内容组装机制 (自然语言解析)
在前端 UI 中，AI 输出内容的组装是一个流式累积、实时解析和组件分层渲染的过程。
流式累积 (Data Accumulation):
文件: frontend/hooks/useChat.ts
逻辑: sendMessageStream 函数通过 llmService 建立流式连接。当收到 chunk（数据块）时，它不会替换整个消息，而是将 chunk.text 追加到当前消息的 content 字段中。如果是工具调用（如搜索）返回的元数据（groundingMetadata），也会实时合并到消息对象中。
状态更新: 使用 setMessages 实时更新 React 状态，这会触发界面重新渲染，让用户看到打字机效果。
实时解析与分离 (Parsing & Separation):
文件: frontend/hooks/useMessageProcessor.ts
逻辑: 这是一个自定义 Hook，专门用于处理原始的消息字符串。它使用正则表达式或字符串查找来识别 <thinking> 和 </thinking> 标签。
分离: 它将原始文本拆分为 thinkingContent（思考过程）和 displayContent（最终回复）。如果标签未闭合（正在生成中），它会将当前所有内容视为思考内容，直到闭合标签出现。
组件分层渲染 (Layered Rendering):
文件: frontend/components/chat/MessageItem.tsx
逻辑: 该组件作为容器，根据解析后的数据按顺序渲染子组件：
搜索过程: SearchProcess.tsx (基于 groundingMetadata)。
思维链: ThinkingBlock.tsx (基于 thinkingContent，支持折叠/展开)。
正文: MarkdownRenderer.tsx (基于 displayContent，处理 Markdown、代码高亮和流式光标)。
附件/媒体: AttachmentGrid.tsx (基于 message.attachments，渲染图片、视频或音频)。
引用源: GroundingSources.tsx (基于搜索来源链接)。

    流程详解与文件对应
数据累积 (useChat.ts):
动作: 监听 llmService.sendMessageStream 返回的 chunk。
逻辑: 将零散的字符流 (chunk.text) 拼接到 accumulatedText，如果有附件（如生成的图片 URL），则推入 accumulatedAttachments。
结果: 更新 React State messages，触发组件重绘。
逻辑解析 (useMessageProcessor.ts):
动作: 这是一个 Custom Hook，在 MessageItem 中调用。
逻辑:
思维链: 检测 <thinking> 标签。如果检测到，将其内容提取给 thinkingContent，并从主要显示内容中移除。
PDF模式: 如果当前是 PDF 提取模式，尝试 JSON.parse 消息内容。
元数据: 提取 groundingMetadata 用于显示搜索来源。
UI 渲染 (MessageItem.tsx):
动作: 作为容器组件，按垂直顺序堆叠子组件。
分层:
顶部: SearchProcess (搜索动画)。
上部: ThinkingBlock (折叠的思考过程)。
中部:
普通模式: MarkdownRenderer (渲染文本、代码块、LaTeX)。
PDF模式: PdfExtractionResult (渲染结构化数据)。
下部: GroundingSources (Google 搜索引用链接)。
底部: AttachmentGrid (生成的图片、视频或音频文件)。
